> 本文档基于第三次虚拟机搭建的实际结果。
> 前两次经历: 野火 Ubuntu 18.04 (VSCode 版本问题放弃) → Ubuntu 24.04 (磁盘满损坏) → 当前 Ubuntu 24.04 全新安装。

## 1.1 环境概述

本项目采用**两阶段开发**策略:

```
阶段一 (虚拟机开发):
  Ubuntu 虚拟机 → 本地编译 (x86) → 用视频文件模拟摄像头 → 开发全部软件功能

阶段二 (开发板部署):
  Ubuntu 虚拟机 → 交叉编译 (Linaro gcc 7.5.0) → 部署到 STM32MP157
```

## 1.6 交叉编译版 OpenCV 3.4.16 for ARM ✅

使用 Linaro gcc 7.5.0 编译, 确保 glibc 兼容。

```
安装路径: /opt/opencv_arm/
toolchain 文件: /home/chris/opencv_arm_toolchain.cmake
验证: ELF 32-bit LSB shared object, ARM ✓
GLIBC 依赖: 仅 2.4~2.6, 开发板完全兼容 ✓
```

编译参数:
```bash
cmake .. \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_INSTALL_PREFIX=/opt/opencv_arm \
    -DCMAKE_TOOLCHAIN_FILE=~/opencv_arm_toolchain.cmake \
    -DENABLE_NEON=ON -DENABLE_VFPV3=ON \
    -DBUILD_TESTS=OFF -DBUILD_PERF_TESTS=OFF -DBUILD_EXAMPLES=OFF \
    -DWITH_GTK=OFF -DWITH_QT=OFF -DWITH_V4L=ON \
    -DWITH_FFMPEG=OFF -DWITH_OPENCL=OFF -DWITH_IPP=OFF
```

> 开发板上已部署: 17 个 .so 文件 → /usr/lib/, ldconfig 已执行, 无需 LD_LIBRARY_PATH。

## 1.10 网络与文件传输 ✅

### 网络互通

```
虚拟机: 192.168.0.130 (桥接模式)
开发板: 192.168.0.43 (WiFi, 延迟 40~467ms 波动较大)
```

### SSH 连接

```bash
ssh debian@192.168.0.43    # 密码: temppwd
su -                        # 切换 root, 密码: root
```

> root SSH 直接登录被禁止, 必须先用 debian 用户登录再 su。

### NFS 共享目录 ✅

```
虚拟机共享目录: /home/chris/nfs_share
开发板挂载点: /mnt/nfs
```

```bash
# 开发板上挂载
mount -t nfs 192.168.0.130:/home/chris/nfs_share /mnt/nfs -o nolock
```

### SCP 文件传输

```bash
scp file debian@192.168.0.43:/tmp/
```

## 1.14 ⚠️ 注意事项

1. **所有交叉编译必须用 Linaro gcc 7.5.0**, 不能用 apt 装的交叉编译器 (glibc 不兼容)
2. **OpenCV 编译前确认磁盘空间充足** (上次就是磁盘满导致虚拟机损坏)
3. **VM_Share 是共享文件夹**, 大文件操作 (如编译) 不要在这里进行, 用 /home/chris/ 或 /opt/
4. **pkg-config 名称**: Ubuntu 24.04 的 OpenCV 是 4.x, 名称为 `opencv4` (不是 `opencv`)
