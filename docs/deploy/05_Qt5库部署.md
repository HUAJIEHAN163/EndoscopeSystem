# Step 5：Qt5 库部署到开发板 [✅ 已完成]

## 这一步在做什么

开发板上没有预装 Qt5 库，但我们的程序使用 Qt5 做界面（按钮、滑块、布局、信号槽等）。需要把虚拟机上的 Qt5 ARM 交叉编译版库文件部署到开发板上。

项目只用到 3 个 Qt5 模块（CMakeLists.txt 中 find_package 指定的）：

| 模块 | 库文件 | 用途 |
|------|--------|------|
| Qt5::Core | libQt5Core.so | 核心库（信号槽、事件循环、QString 等） |
| Qt5::Gui | libQt5Gui.so | 图形库（QImage、QPainter、字体等） |
| Qt5::Widgets | libQt5Widgets.so | 控件库（QPushButton、QSlider、布局等） |

---

## 5.1 虚拟机上：复制库文件到 NFS 共享目录

```bash
mkdir -p /home/chris/nfs_share/qt5_libs
cp /opt/qt-everywhere-src-5.11.3/lib/libQt5Core.so* /home/chris/nfs_share/qt5_libs/
cp /opt/qt-everywhere-src-5.11.3/lib/libQt5Gui.so* /home/chris/nfs_share/qt5_libs/
cp /opt/qt-everywhere-src-5.11.3/lib/libQt5Widgets.so* /home/chris/nfs_share/qt5_libs/
```

---

## 5.2 开发板上：挂载 NFS 并复制库文件

```bash
ssh debian@192.168.0.43
su - root
# 密码：root

mount -t nfs 192.168.0.130:/home/chris/nfs_share /mnt/nfs -o nolock
cp /mnt/nfs/qt5_libs/libQt5*.so* /usr/lib/
```

---

## 5.3 开发板上：修复符号链接

`cp` 复制会把符号链接变成实体文件，导致 ldconfig 警告。需要删除后重建：

```bash
cd /usr/lib

# 删除多余的实体文件（只保留 .so.5.11.3）
rm libQt5Core.so libQt5Core.so.5 libQt5Core.so.5.11
rm libQt5Gui.so libQt5Gui.so.5 libQt5Gui.so.5.11
rm libQt5Widgets.so libQt5Widgets.so.5 libQt5Widgets.so.5.11

# 重建符号链接（链式指向）
ln -s libQt5Core.so.5.11.3 libQt5Core.so.5.11
ln -s libQt5Core.so.5.11 libQt5Core.so.5
ln -s libQt5Core.so.5 libQt5Core.so

ln -s libQt5Gui.so.5.11.3 libQt5Gui.so.5.11
ln -s libQt5Gui.so.5.11 libQt5Gui.so.5
ln -s libQt5Gui.so.5 libQt5Gui.so

ln -s libQt5Widgets.so.5.11.3 libQt5Widgets.so.5.11
ln -s libQt5Widgets.so.5.11 libQt5Widgets.so.5
ln -s libQt5Widgets.so.5 libQt5Widgets.so
```

## 5.4 开发板上：刷新库缓存并验证

```bash
ldconfig
ls -l /usr/lib/libQt5Core.so*
```

正确结果：
```
libQt5Core.so → libQt5Core.so.5
libQt5Core.so.5 → libQt5Core.so.5.11
libQt5Core.so.5.11 → libQt5Core.so.5.11.3
libQt5Core.so.5.11.3  ← 实体文件 (5.5MB)
```

---

## 5.5 符号链接原理说明

Linux 动态库用符号链接实现版本管理：

```
libQt5Core.so        ← 编译时链接用（不带版本号）
libQt5Core.so.5      ← 运行时加载用（主版本号）
libQt5Core.so.5.11   ← 次版本号
libQt5Core.so.5.11.3 ← 实体文件（完整版本号）
```

好处：升级库时只需替换实体文件并更新符号链接，不需要重新编译程序。

---

## 5.6 为什么 cp 会破坏符号链接

```bash
# 虚拟机上的原始状态（符号链接）
libQt5Core.so → libQt5Core.so.5 → libQt5Core.so.5.11 → libQt5Core.so.5.11.3

# cp 复制后（全变成实体文件，每个都是 5.5MB 的完整拷贝）
libQt5Core.so       5.5MB  ← 实体文件
libQt5Core.so.5     5.5MB  ← 实体文件（应该是链接）
libQt5Core.so.5.11  5.5MB  ← 实体文件（应该是链接）
libQt5Core.so.5.11.3 5.5MB ← 实体文件

# ldconfig 发现 .so.5 不是符号链接，报警告
```

解决方法就是删掉多余的实体文件，手动重建符号链接。
