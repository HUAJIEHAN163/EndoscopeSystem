# Step 1：虚拟机编译环境搭建 [✅ 已完成]

## 这一步在做什么

在虚拟机上搭建交叉编译环境，让 x86 电脑能编译出 ARM 开发板可运行的程序。

交叉编译的含义：
```
普通编译：x86 电脑编译 → x86 程序（在电脑上运行）
交叉编译：x86 电脑编译 → ARM 程序（在开发板上运行）
```

开发板 CPU 性能弱，直接在上面编译很慢，所以在虚拟机上编译好再传过去。

## 1.2 Linaro gcc 7.5.0 交叉编译器

**为什么不用 apt 装的交叉编译器？**

| 编译器 | sysroot glibc | 开发板 glibc | 兼容？ |
|--------|--------------|-------------|--------|
| apt gcc-13 | 2.39 | 2.28 | ❌ 不兼容，运行时报错 |
| Linaro gcc 7.5.0 | 2.25 | 2.28 | ✅ 兼容（低版本兼容高版本） |

**安装步骤：**

```bash
# 1. 从 Linaro 官网下载预编译包
#    地址：https://releases.linaro.org/components/toolchain/binaries/7.5-2019.12/arm-linux-gnueabihf/
#    文件：gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf.tar.xz
wget https://releases.linaro.org/components/toolchain/binaries/7.5-2019.12/arm-linux-gnueabihf/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf.tar.xz

# 2. 解压到 /opt/toolchain/（Linaro 工具链是预编译的，解压即用，不需要编译）
sudo mkdir -p /opt/toolchain
sudo tar -xf gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf.tar.xz -C /opt/toolchain/

# 3. 验证
/opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf/bin/arm-linux-gnueabihf-gcc --version
# 输出：arm-linux-gnueabihf-gcc (Linaro GCC 7.5-2019.12) 7.5.0
```

安装路径：`/opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf/`

目录结构：
```
gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf/
├── bin/          # arm-linux-gnueabihf-gcc、g++、ld 等可执行文件
├── arm-linux-gnueabihf/  # sysroot（目标平台的头文件和库）
├── lib/          # 宿主机运行时库
└── gcc-linaro-7.5.0-2019.12-linux-manifest.txt  # 版本清单
```

## 1.3 OpenCV 3.4.16 ARM 交叉编译版

用 Linaro gcc 7.5.0 从源码编译的 ARM 版 OpenCV。

| 项目 | 值 |
|------|-----|
| 源码 | gitee 镜像 OpenCV 3.4.16 |
| 编译工具链 | Linaro gcc 7.5.0 |
| 安装路径 | /opt/opencv_arm/ |
| 验证 | ELF 32-bit ARM ✓，GLIBC 依赖 2.4~2.6 ✓ |

**编译步骤：**

```bash
# 1. 下载源码（gitee 镜像，比 github 快）
git clone https://gitee.com/mirrors/opencv.git -b 3.4.16 --depth=1
cd opencv

# 2. 创建交叉编译用的 toolchain 文件
cat > /tmp/opencv_arm_toolchain.cmake << 'EOF'
set(CMAKE_SYSTEM_NAME Linux)
set(CMAKE_SYSTEM_PROCESSOR arm)
set(TOOLCHAIN /opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf)
set(CMAKE_C_COMPILER   ${TOOLCHAIN}/bin/arm-linux-gnueabihf-gcc)
set(CMAKE_CXX_COMPILER ${TOOLCHAIN}/bin/arm-linux-gnueabihf-g++)
set(CMAKE_FIND_ROOT_PATH_MODE_PROGRAM NEVER)
set(CMAKE_FIND_ROOT_PATH_MODE_LIBRARY ONLY)
set(CMAKE_FIND_ROOT_PATH_MODE_INCLUDE ONLY)
EOF

# 3. 在虚拟机本地目录编译（不要在 VM_Share 下编译，会出现 I/O 错误）
mkdir -p ~/build_opencv_arm && cd ~/build_opencv_arm

cmake /path/to/opencv \
    -DCMAKE_TOOLCHAIN_FILE=/tmp/opencv_arm_toolchain.cmake \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_INSTALL_PREFIX=/opt/opencv_arm \
    -DBUILD_SHARED_LIBS=ON \
    -DWITH_NEON=ON \
    -DENABLE_NEON=ON \
    -DWITH_GTK=OFF \
    -DWITH_QT=OFF \
    -DWITH_FFMPEG=OFF \
    -DWITH_GSTREAMER=OFF \
    -DBUILD_TESTS=OFF \
    -DBUILD_PERF_TESTS=OFF \
    -DBUILD_EXAMPLES=OFF \
    -DBUILD_opencv_python2=OFF \
    -DBUILD_opencv_python3=OFF

make -j$(nproc)
sudo make install
```

**关键编译选项说明：**

| 选项 | 值 | 原因 |
|---|---|---|
| `CMAKE_INSTALL_PREFIX` | `/opt/opencv_arm` | 和 x86 版隔离，不冲突 |
| `WITH_NEON=ON` | ON | 开启 Cortex-A7 NEON 加速 |
| `WITH_GTK/QT/FFMPEG` | OFF | 开发板上没有这些库 |
| `BUILD_TESTS/EXAMPLES` | OFF | 减少编译时间 |
| `BUILD_SHARED_LIBS` | ON | 生成 .so 动态库，减小可执行文件体积 |

**验证：**

```bash
file /opt/opencv_arm/lib/libopencv_core.so.3.4.16
# 输出：ELF 32-bit LSB shared object, ARM, EABI5 version 1 (GNU/Linux)
```

## 1.4 Qt5.11.3 ARM 交叉编译版

使用野火预编译的 ARM 版，从 VM_Share 解压。

| 项目 | 值 |
|------|-----|
| 来源 | VM_Share/qt-everywhere-src-5.11.3.tar.gz（野火预编译） |
| 安装路径 | /opt/qt-everywhere-src-5.11.3/ |
| qmake | x86-64 可执行文件（在虚拟机上运行，生成 ARM 的 Makefile） |
| 库文件 | ELF 32-bit ARM ✓ |
| GLIBC 依赖 | 最高 2.28，开发板刚好兼容 ✓ |

**安装步骤：**

```bash
# 直接解压预编译包，不需要重新编译
sudo tar -xf /mnt/hgfs/VM_Share/qt-everywhere-src-5.11.3.tar.gz -C /opt/

# 验证 qmake
/opt/qt-everywhere-src-5.11.3/bin/qmake --version
# 输出：QMake version 3.1, Using Qt version 5.11.3

# 验证库文件是 ARM 的
file /opt/qt-everywhere-src-5.11.3/lib/libQt5Core.so.5.11.3
# 输出：ELF 32-bit LSB shared object, ARM
```

> 注：这个包是野火针对 STM32MP157 预编译的，已内置 Linaro 工具链配置。如果需要自己从源码编译 Qt，参考野火官方文档。

## 1.5 ARM 工具链文件

项目中的 `scripts/arm_toolchain.cmake` 告诉 CMake 使用 Linaro 交叉编译器：

```cmake
set(TOOLCHAIN_ROOT /opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf)
set(CMAKE_C_COMPILER   ${TOOLCHAIN_ROOT}/bin/arm-linux-gnueabihf-gcc)
set(CMAKE_CXX_COMPILER ${TOOLCHAIN_ROOT}/bin/arm-linux-gnueabihf-g++)

# Cortex-A7 NEON 优化
set(CMAKE_C_FLAGS   "-march=armv7-a -mfpu=neon-vfpv4 -mfloat-abi=hard")
set(CMAKE_CXX_FLAGS "-march=armv7-a -mfpu=neon-vfpv4 -mfloat-abi=hard")
```

以下是arm_toolchain.cmake的原文：
# ARM 交叉编译工具链 (STM32MP157)
# 使用方法: cmake -DCMAKE_TOOLCHAIN_FILE=scripts/arm_toolchain.cmake ..
#
# 必须使用 Linaro gcc 7.5.0 (sysroot glibc 2.25)
# apt 的交叉编译器 glibc 2.39 与开发板 glibc 2.28 不兼容!

set(CMAKE_SYSTEM_NAME Linux)
set(CMAKE_SYSTEM_PROCESSOR arm)

set(TOOLCHAIN_ROOT /opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf)
set(CMAKE_C_COMPILER   ${TOOLCHAIN_ROOT}/bin/arm-linux-gnueabihf-gcc)
set(CMAKE_CXX_COMPILER ${TOOLCHAIN_ROOT}/bin/arm-linux-gnueabihf-g++)

set(CMAKE_FIND_ROOT_PATH_MODE_PROGRAM NEVER)
set(CMAKE_FIND_ROOT_PATH_MODE_LIBRARY ONLY)
set(CMAKE_FIND_ROOT_PATH_MODE_INCLUDE ONLY)

# Cortex-A7 NEON 优化
set(CMAKE_C_FLAGS   "${CMAKE_C_FLAGS} -march=armv7-a -mfpu=neon-vfpv4 -mfloat-abi=hard" CACHE STRING "")
set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -march=armv7-a -mfpu=neon-vfpv4 -mfloat-abi=hard" CACHE STRING "")
