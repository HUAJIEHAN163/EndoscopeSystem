# Step 7：部署并运行 [✅ 已完成]

## 这一步在做什么

把交叉编译好的程序通过 NFS 传到开发板，运行内窥镜程序，验证摄像头采集和界面显示。

---

## 7.1 开发板上挂载 NFS

```bash
ssh debian@192.168.0.43
su - root
# 密码：root

mount -t nfs 192.168.0.130:/home/chris/nfs_share /mnt/nfs -o nolock
```

---

## 7.2 运行程序

```bash
cd /mnt/nfs

# 确认摄像头设备存在
ls /dev/video*

# 运行
./endoscope
```

程序会自动检测 `/dev/video0`，有摄像头就用 V4L2 采集，没有就降级为文件模式。

## 7.4 可能遇到的问题

| 问题 | 可能原因 | 排查方法 |
|------|---------|---------|
| 找不到共享库 | Qt5/OpenCV 库没部署 | `ldd ./endoscope` 查看缺哪个库 |
| 无法打开 /dev/video0 | 权限不足 | 用 root 运行，或 `chmod 666 /dev/video0` |
| 没有显示界面 | 没有显示环境 | 需要在开发板的屏幕上运行，不能在 SSH 终端 |
| 画面全黑 | 摄像头没接好 | `v4l2-ctl -d /dev/video0 --list-ctrls` 确认设备正常 |
| 帧率很低 | CPU 处理不过来 | 先关闭所有算法，看原始帧率 |
| 段错误 | glibc 不兼容 | `readelf -V ./endoscope` 检查 GLIBC 依赖 |

### 关于显示环境

Qt 程序需要显示环境才能显示界面：
- 如果开发板接了屏幕且运行了桌面环境 → 直接在开发板屏幕上操作
- 如果通过 SSH 登录 → 需要设置 DISPLAY 环境变量或使用 framebuffer：

```bash
# 方式 1：使用 linuxfb（直接写 framebuffer，不需要 X11）
./endoscope -platform linuxfb

# 方式 2：使用 eglfs（OpenGL ES，性能更好）
./endoscope -platform eglfs
```

## 7.5 快速排查命令

```bash
# 检查缺少哪些动态库
ldd ./endoscope

# 检查摄像头设备
v4l2-ctl -d /dev/video0 --list-ctrls

# 检查内核日志（摄像头驱动相关）
dmesg | grep -i video
dmesg | grep -i ov5640

# 检查 CPU 占用
top
```
