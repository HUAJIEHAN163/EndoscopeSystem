# USM锐化文字模糊问题分析

## 问题描述

使用USM锐化处理 `test_text.jpg` 时，文字边缘反而变模糊了。

## 测试结果

```
原图清晰度: 967.214
锐化后清晰度: 2978.43
清晰度提升: 207.94%
```

虽然整体清晰度提升了207%，但文字边缘出现了问题。

## 根本原因：削波效应（Clipping）

### 边缘像素值变化

```
位置  | 原图值 | 模糊值 | 锐化值 | 说明
------|--------|--------|--------|------
x=55  | 180    | 156    | 204    | 正常增强
x=56  | 48     | 99     | 0      | ❌ 削波到0！
x=57  | 49     | 59     | 39     | 反而变暗
```

### 削波原理

USM锐化公式：`result = src + amount × (src - blurred)`

对于 x=56：
```
result = 48 + 1.0 × (48 - 99)
       = 48 - 51
       = -3
       → 削波到 0（因为像素值范围是0-255）
```

**削波破坏了边缘的灰度过渡，导致视觉上的"模糊感"或"生硬感"。**

## 为什么内窥镜图像没问题？

1. **内窥镜图像对比度较低**
   - 组织边缘的梯度变化相对平缓
   - 不容易出现极端的锐化值

2. **文字图像对比度极高**
   - 文字边缘：180 vs 48（差值132）
   - 锐化后容易超出0-255范围

## 解决方案

### 方案1：降低锐化强度（推荐）

```cpp
// 对于高对比度图像，使用较小的amount
double amount = 0.5;  // 而不是1.0
```

### 方案2：使用饱和运算

OpenCV的 `cv::addWeighted()` 已经自动做了饱和处理，但可以改用更温和的混合：

```cpp
// 限制锐化强度，避免过度
cv::Mat unsharpMask;
cv::subtract(src, blurred, unsharpMask);
cv::addWeighted(src, 1.0, unsharpMask, 0.5, 0, dst);  // amount=0.5
```

### 方案3：自适应锐化强度

根据局部对比度调整锐化强度：

```cpp
// 高对比度区域使用较小的amount
// 低对比度区域使用较大的amount
```

## 结论

1. **USM锐化算法本身没有bug**，公式是正确的
2. **文字模糊是削波效应导致的**，这是USM锐化的固有特性
3. **解决方法**：
   - 降低锐化强度（amount从1.0降到0.3-0.5）
   - 或者对文字图像不使用锐化
   - 或者使用自适应锐化（只锐化真实边缘，避免过度锐化）

## 建议

对于内窥镜系统：
- **内窥镜图像**：使用 amount=0.8-1.0（效果好）
- **高对比度图像**：使用 amount=0.3-0.5（避免削波）
- **或者统一使用自适应锐化**：自动避免过度锐化问题
