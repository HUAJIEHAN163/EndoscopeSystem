## 二、集成方案设计

### 2.1 方案选择

根据项目架构（参考docs/13_4_实时处理与离线处理分离），有两种集成方案：

#### 方案A：替换现有USM锐化（推荐）

**优点**：
- 简单直接，不增加UI复杂度
- 用户体验一致（仍然是一个"锐化"开关）
- 效果更好（自适应避免过度锐化）

**缺点**：
- 性能略慢（10ms vs 7ms）
- 无法A/B对比两种算法

**适用场景**：
- 追求最佳效果
- 用户不需要选择算法

#### 方案B：新增独立开关（灵活）

**优点**：
- 保留USM和自适应两种选择
- 可以A/B对比效果
- 灵活性高

**缺点**：
- UI更复杂（两个锐化开关）
- 用户可能困惑

**适用场景**：
- 开发测试阶段
- 需要对比效果

### 2.2 推荐方案：方案A（替换USM）

基于以下理由：
1. 自适应锐化效果更好（避免平滑区域过度锐化）
2. 性能可接受（10ms，仍在实时范围内）
3. UI简洁，用户体验好

```cpp
#ifdef ADAPTIVE_SHARPEN_OPTIMIZED
    // =================================================================
    // 自适应锐化 - 性能优化版 v2.0
    // =================================================================
    
    // 主函数：性能优化版（修复bug + NEON + 内存复用）
    // 参数：
    //   src       — 输入图像（BGR）
    //   dst       — 输出图像（BGR）
    //   amount    — 锐化强度（0.1-5.0），推荐0.8
    //   threshold — 边缘阈值（0-255），推荐20
    static void applyAdaptiveSharpenOpt(const cv::Mat &src, cv::Mat &dst,
                                       double amount = 0.8, int threshold = 20);
    
    // NEON混合函数（公开以便测试）
    static void blendWithMaskNEON(const uchar* src, const uchar* usm,
                                  const uchar* mask, uchar* dst, int total_pixels);

private:
    // 预分配缓冲区（线程局部存储）
    struct AdaptiveSharpenBuffers {
        cv::Mat gray, lap, mask, blurred, usm;
        bool initialized = false;
        
        void init(const cv::Size& size) {
            if (!initialized || gray.size() != size) {
                gray.create(size, CV_8UC1);
                lap.create(size, CV_16S);
                mask.create(size, CV_8UC1);
                blurred.create(size, CV_8UC3);
                usm.create(size, CV_8UC3);
                initialized = true;
            }
        }
    };
    
    static thread_local AdaptiveSharpenBuffers s_buffers;
#endif
```

## 五、常见问题排查

### 5.1 编译错误

**问题1：NEON指令未定义**
```
error: 'vld1_u8' was not declared in this scope
```

**解决**：
```cmake
# CMakeLists.txt - 确保启用NEON
set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -mfpu=neon")
```

**问题2：thread_local不支持**
```
error: 'thread_local' does not name a type
```

**解决**：
```cmake
# CMakeLists.txt - 确保C++11支持
set(CMAKE_CXX_STANDARD 11)
```

### 5.2 运行时错误

**问题1：性能不达标（>15ms）**

**排查**：
1. 检查是否启用了NEON（qDebug输出）
2. 检查是否使用了预分配缓冲区
3. 检查是否有不必要的内存拷贝

**解决**：
- 参考《性能优化版方案_v2.0.md》第二章优化方案
- 使用性能分析工具（perf）定位瓶颈

**问题2：效果不理想**

**排查**：
1. 检查参数设置（amount、threshold）
2. 检查mask是否正确生成
3. 检查是否有bug（LUT查找）

**解决**：
- 调整参数：amount=0.5-1.0, threshold=10-30
- 使用测试图像验证算法正确性

### 5.3 UI问题

**问题1：开关不生效**

**排查**：
1. 检查displayTimer中是否同步了config
2. 检查process()中是否调用了算法
3. 检查宏定义是否正确

**解决**：
```cpp
// mainwindow.cpp - displayTimer
qDebug() << "sharpen:" << config.sharpen 
         << "adaptive:" << config.sharpenAdaptive;

// imageprocessor.cpp - process()
qDebug() << "调用自适应锐化";
```
