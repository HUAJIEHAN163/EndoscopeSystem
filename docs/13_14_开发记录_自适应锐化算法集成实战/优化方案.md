算法选择：文档用 Laplacian/Sobel 生成掩码并与 USM 混合，思路合理，能获得边缘自适应锐化；建议始终在亮度通道（Y）上处理，避免色彩伪影。
精度与性能折中：快速方案用浮点实现利于验证；性能版本应用定点（CV_16S/CV_8U）或缩放整型比例避免大量浮点操作。
掩码归一化：不要每像素除法（慢）；用预计算查表（LUT）或固定点乘法近似归一化，能显著降耗。
高斯模糊：优先使用 OpenCV 内置的 separable GaussianBlur（已优化）；若核很小，考虑用 3x3 或 box-approximation 以更快响应。
临时内存与拷贝：消除不必要的临时 Mat，使用预分配/对象池重用缓冲区（输入→工作稿→输出三缓冲原则）。尽量在零拷贝管线中复用来自 v4l2 的映射内存。
并行化：对像素混合、掩码计算与卷积使用 cv::parallel_for_ 或行级并行（不要在采集线程中阻塞 UI）。在 Qt 框架内优先使用现有线程池/任务框架一致性。
向量化（NEON）：先通过剖面确定热点（占比>30%）；对混合/归一化循环实现 NEON，优先优化内存访问对齐和连续行处理以提高效率。文档建议正确。
算法变体：可试两种掩码计算——Laplacian 方差（纹理强度）与 Sobel magnitude（边缘位置），并对比视觉与耗时。还可考虑采用 bilateral/CLAHE 组合，但作为后续可选项（复杂且耗时）。
参数搜索与自动化：添加批量参数扫描工具（grid search），收集 Laplacian/SSIM 与时间，自动输出 Pareto 前沿（速度 vs 清晰度）。
验证对比基线：对比时确保 USM 与自适应锐化使用相同的 input color-space、分辨率与相同的“amount/threshold”语义，以保证公平。
