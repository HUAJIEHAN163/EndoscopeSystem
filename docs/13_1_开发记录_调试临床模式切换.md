# 开发记录：调试模式 / 临床模式切换 + 预设参数管理

**日期**：2025-01  
**对应优化计划**：P0  

---

## 一、需求背景

当前界面所有算法参数以复选框和滑块形式暴露，适合开发人员调参，但不适合医生使用。实际产品中需要区分两种工作模式：

- **调试模式（工程师用）**：保留所有参数控件，用于算法调优和参数导出
- **临床模式（医生用）**：隐藏参数细节，医生只需选择检查部位（胃镜/肠镜/支气管镜），一键加载预设参数

---

## 二、整体方案

### 2.1 架构设计

```
┌─────────────────────────────────────────────────────┐
│                    MainWindow                        │
│  ┌──────────┐  ┌──────────────────────────────────┐ │
│  │          │  │ [切换模式按钮]                     │ │
│  │          │  │ ┌────────────────────────────────┐│ │
│  │  视频    │  │ │  QStackedWidget               ││ │
│  │  显示    │  │ │  ┌──────────┐ ┌─────────────┐ ││ │
│  │  区域    │  │ │  │调试面板   │ │临床面板      │ ││ │
│  │          │  │ │  │复选框    │ │预设下拉框    │ ││ │
│  │          │  │ │  │滑块      │ │             │ ││ │
│  │          │  │ │  │导出按钮  │ │             │ ││ │
│  │          │  │ │  └──────────┘ └─────────────┘ ││ │
│  │          │  │ └────────────────────────────────┘│ │
│  │          │  │ [操作按钮: 拍照/录像/冻结/旋转]    │ │
│  │          │  │ [状态信息: FPS / 状态]             │ │
│  └──────────┘  └──────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

### 2.2 核心思路

- 使用 `QStackedWidget` 实现调试面板和临床面板的切换，两个面板叠放在同一位置，同一时间只显示一个
- 预设参数以 JSON 文件存储在 `config/presets/` 目录下，程序启动时自动加载
- 新增 `PresetManager` 类负责 JSON 文件的读写，与界面逻辑解耦
- 操作按钮（拍照/录像/冻结/旋转）和状态信息两种模式共用，不重复创建
- 调试模式下新增"导出为预设"按钮，可将当前调好的参数保存为新的预设文件

### 2.3 数据流变化

```
原来：界面控件 → 每帧读取控件状态 → processFrame

现在：
  调试模式：界面控件 → 每帧读取控件状态 → processFrame（不变）
  临床模式：预设选择 → m_procConfig 一次性设置 → processFrame 直接使用
```

## 三、新增文件

### 3.1 `src/ui/presetmanager.h`

预设管理器头文件，声明三个静态方法：

```cpp
class PresetManager {
public:
    // 从目录加载所有 .json 预设
    static QMap<QString, ImageProcessor::Config> loadAll(const QString &dir);
    // 加载单个预设
    static bool load(const QString &path, QString &name, ImageProcessor::Config &cfg);
    // 保存预设
    static bool save(const QString &path, const QString &name, const ImageProcessor::Config &cfg);
};
```

### 3.2 `src/ui/presetmanager.cpp`

预设管理器实现：
- `loadAll()`：遍历目录下所有 `.json` 文件，逐个调用 `load()` 解析
- `load()`：用 `QJsonDocument` 解析 JSON，将字段映射到 `ImageProcessor::Config` 结构体
- `save()`：将 `Config` 结构体序列化为 JSON，写入文件（`QJsonDocument::Indented` 格式化输出）

### 3.3 `config/presets/gastroscope.json`

胃镜模式预设参数：
- 白平衡 ✓、CLAHE ✓（clipLimit=3.0）、去雾 ✓（omega=0.85）
- 锐化 ✓（amount=1.2）、降噪 ✓（d=5）

### 3.4 `config/presets/colonoscope.json`

肠镜模式预设参数：
- 白平衡 ✓、CLAHE ✓（clipLimit=4.0）、去雾 ✗
- 锐化 ✓（amount=1.8）、降噪 ✓（d=7）

### 3.5 `config/presets/bronchoscope.json`

支气管镜模式预设参数：
- 白平衡 ✓、CLAHE ✓（clipLimit=2.5）、去雾 ✓（omega=0.90）
- 锐化 ✗、降噪 ✓（d=9）

### 3.6 预设 JSON 格式说明

```json
{
    "name": "预设显示名称",
    "whiteBalance": true/false,
    "clahe": true/false,
    "claheClipLimit": 3.0,
    "undistort": true/false,
    "dehaze": true/false,
    "dehazeOmega": 0.95,
    "dehazeRadius": 7,
    "sharpen": true/false,
    "sharpenAmount": 1.5,
    "denoise": true/false,
    "denoiseD": 5,
    "edgeDetect": true/false,
    "threshold": true/false,
    "thresholdValue": 128
}
```

添加新预设只需在 `config/presets/` 下新建 JSON 文件，程序启动时自动识别。

## 四、修改文件

### 4.1 `src/ui/mainwindow.h`

**新增头文件引入：**
- `QStackedWidget`：面板切换容器
- `QComboBox`：预设下拉框

**新增槽函数：**
- `onSwitchMode()`：切换调试/临床模式
- `onPresetSelected(int index)`：选择预设时加载对应参数
- `onExportPreset()`：导出当前参数为预设文件

**新增私有方法：**
- `loadPresets()`：从 `config/presets/` 加载所有预设
- `applyConfig(const Config &cfg)`：将 Config 同步到界面控件

**新增成员变量：**

| 变量 | 类型 | 说明 |
|------|------|------|
| `m_clinicalMode` | `bool` | 当前模式标志 |
| `m_btnSwitchMode` | `QPushButton*` | 模式切换按钮 |
| `m_ctrlStack` | `QStackedWidget*` | 面板切换容器 |
| `m_debugPanel` | `QWidget*` | 调试模式面板 |
| `m_clinicalPanel` | `QWidget*` | 临床模式面板 |
| `m_cboPreset` | `QComboBox*` | 预设下拉框 |
| `m_btnExportPreset` | `QPushButton*` | 导出预设按钮 |
| `m_presetNames` | `QStringList` | 预设名称列表 |
| `m_presetConfigs` | `QList<Config>` | 预设参数列表 |

### 4.2 `src/ui/mainwindow.cpp`

**构造函数变更：**
- 初始化列表新增 `m_clinicalMode(false)`
- 新增 `loadPresets()` 调用（在 `setupUi()` 之后）

**setupUi() 重构：**
- 右侧面板改为 `QVBoxLayout`，顶部放模式切换按钮
- 原有的算法控件（复选框、滑块）移入 `m_debugPanel`（QStackedWidget index 0）
- 新建 `m_clinicalPanel`（QStackedWidget index 1），包含预设下拉框
- 调试面板新增"导出为预设"按钮
- 操作按钮和状态信息放在 QStackedWidget 外部，两种模式共用

**processFrame() 变更：**
- 调试模式：从界面控件读取参数（原有逻辑不变）
- 临床模式：直接使用 `m_procConfig`（已在 `onPresetSelected` 中设置好）

**新增函数：**
- `loadPresets()`：调用 `PresetManager::loadAll()` 加载预设，填充下拉框
- `onSwitchMode()`：切换 `QStackedWidget` 的 currentIndex，更新按钮文字
- `onPresetSelected(int)`：将选中预设的 Config 赋值给 `m_procConfig`
- `onExportPreset()`：弹出输入框获取名称，调用 `PresetManager::save()` 保存，然后重新加载预设列表
- `applyConfig(const Config&)`：将 Config 值同步到界面复选框和滑块

## 六、操作说明

| 操作 | 说明 |
|------|------|
| 切换模式 | 点击右上角"切换到临床模式"/"切换到调试模式"按钮 |
| 选择预设 | 临床模式下，从下拉框选择检查模式，参数立即生效 |
| 导出预设 | 调试模式下，调好参数后点击"导出为预设"，输入名称保存 |
| 添加预设 | 在 `config/presets/` 下新建 JSON 文件，重启程序自动加载 |
