# 开发记录：冻结帧清晰度优化

**日期**：2025-01

---

## 一、问题描述

冻结画面时，显示的帧不够清晰，呈现运动模糊（motion blur）效果。

**原因**：摄像头曝光时间内画面或镜头在移动，导致单帧图像模糊。视频连续播放时人眼感觉不到，冻结后单帧放大看就很明显。原有实现直接停在最后一帧，没有做任何筛选。

---

## 二、解决方案

缓存最近 N 帧，冻结时用拉普拉斯方差评估每帧清晰度，选得分最高的帧显示。

### 2.1 原理：拉普拉斯清晰度评估

拉普拉斯算子是二阶导数算子，对图像边缘和细节敏感：
- 清晰图像：边缘锐利，拉普拉斯响应大，方差高
- 模糊图像：边缘平滑，拉普拉斯响应小，方差低

```
清晰度得分 = Var(Laplacian(gray_image))
```

值越大，图像越清晰。

### 2.2 整体流程

```
正常播放时：
  每帧到达 → 处理 → 存入环形缓存（保留最近 8 帧）→ 显示

按下冻结时：
  遍历缓存的 8 帧 → 计算每帧清晰度得分 → 选最高分 → 显示该帧
```

### 3.2 `src/processing/imageprocessor.cpp`

新增清晰度评估方法实现：

```cpp
double ImageProcessor::calcSharpness(const cv::Mat &src) {
    cv::Mat gray, lap;
    if (src.channels() == 3)
        cv::cvtColor(src, gray, cv::COLOR_BGR2GRAY);
    else
        gray = src;
    cv::Laplacian(gray, lap, CV_64F);       // 拉普拉斯算子
    cv::Scalar mu, sigma;
    cv::meanStdDev(lap, mu, sigma);          // 计算均值和标准差
    return sigma.val[0] * sigma.val[0];      // 方差 = 标准差²
}
```

### 3.4 `src/ui/mainwindow.cpp`

**onFrameReady() — 新增帧缓存逻辑：**

```cpp
// 修复前：无缓存，冻结时停在最后一帧
void MainWindow::onFrameReady(const QImage &image) {
    if (m_frozen) return;
    QImage processed = processFrame(image);
    // ... 直接显示 processed
}

// 修复后：缓存最近 8 帧
void MainWindow::onFrameReady(const QImage &image) {
    if (m_frozen) return;
    QImage processed = processFrame(image);

    // 缓存处理后的帧
    m_frameBuffer.push_back(processed.copy());
    if (static_cast<int>(m_frameBuffer.size()) > FRAME_BUFFER_SIZE)
        m_frameBuffer.pop_front();

    // ... 显示 processed
}
```

**onToggleFreeze() — 从缓存中选最清晰帧：**

```cpp
// 修复前：直接冻结当前帧
void MainWindow::onToggleFreeze() {
    m_frozen = !m_frozen;
    m_btnFreeze->setText(m_frozen ? "继续 (Space)" : "冻结 (Space)");
    m_lblStatus->setText(m_frozen ? "画面已冻结" : "运行中");
}

// 修复后：遍历缓存帧，选清晰度最高的
void MainWindow::onToggleFreeze() {
    m_frozen = !m_frozen;

    if (m_frozen && !m_frameBuffer.empty()) {
        double bestScore = -1;
        int bestIdx = 0;
        for (int i = 0; i < static_cast<int>(m_frameBuffer.size()); i++) {
            cv::Mat mat = ImageConvert::qimageToMat(m_frameBuffer[i]);
            if (mat.empty()) continue;
            double score = ImageProcessor::calcSharpness(mat);
            if (score > bestScore) {
                bestScore = score;
                bestIdx = i;
            }
        }
        // 用最清晰帧更新显示
        m_displayImage = m_frameBuffer[bestIdx].scaled(...);
        update();
        m_lblStatus->setText(QString("已冻结 (最佳帧 %1/%2, 清晰度 %3)")
                             .arg(bestIdx + 1).arg(m_frameBuffer.size())
                             .arg(bestScore, 0, 'f', 1));
    } else {
        m_frameBuffer.clear();  // 解冻时清空缓存
    }
}
```

## 四、参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| FRAME_BUFFER_SIZE | 8 | 缓存帧数，8 帧约覆盖 0.27 秒（30fps） |
| Laplacian depth | CV_64F | 双精度计算拉普拉斯，精度高 |
| 评估指标 | 方差 | 标准差的平方，对模糊敏感 |

缓存 8 帧的内存开销：768×576×4(RGBA)×8 ≈ 14MB，可接受。

## 五、效果

- 冻结前：停在最后一帧，可能是运动模糊帧
- 冻结后：从最近 8 帧中自动选择最清晰的一帧，状态栏显示选中的帧序号和清晰度得分

## 六、后续改进方向

- 缓存帧数可配置化（写入 endoscope.conf）
- 在缩小的图像上计算清晰度，进一步减少冻结时的计算延迟
- 结合硬件：缩短曝光时间从源头减少运动模糊
实际产品中主要靠硬件解决：
缩短曝光时间：曝光越短，运动模糊越小，但画面会变暗，需要增大增益或补光
提高帧率：60fps 比 30fps 每帧的运动量减半
闪光灯模式：内窥镜冷光源在曝光瞬间闪一下强光，配合极短曝光，几乎无模糊
光源只在曝光瞬间闪亮，持续时间极短（微秒级）
相当于"闪光灯拍照"，每帧都是短曝光，运动模糊极小
灭灯期间不曝光，所以人眼看视频感觉光在闪

好处：
曝光时间极短，彻底解决运动模糊
光源不用一直全功率工作，减少发热（探头温度是医疗安全红线）
可以配合不同波长光源交替闪烁（白光 + 窄带光），实现 NBI（窄带成像）等特殊观察模式
所以你之前冻结帧模糊的问题，在实际产品中基本不存在——每帧都是"闪光灯拍的"，天然清晰。
这也是为什么内窥镜的光源控制放在 M4 侧用 PWM 做——需要精确到微秒级的时序控制，Linux 做不到这个实时性。
