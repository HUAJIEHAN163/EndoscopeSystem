# 13_11 开发记录：A7 侧 Qt UI 集成

## 一、背景

M4 侧 RPMsg 通信（步骤 6）和 FreeRTOS 多任务（步骤 7）完成后，需要在 A7 侧的 EndoscopeSystem Qt 程序中集成 RPMsg 通信，实现完整的 A7-M4 联动。

---

## 二、架构

```
RpmsgListener (QThread)              MainWindow
├── open("/dev/ttyRPMSG0")          ├── 连接信号
├── select() 轮询读取               │   ├── temperatureUpdated → 状态栏
│   ├── "T255\n" → emit temp(25.5)  │   ├── freezeRequested → onToggleFreeze
│   ├── "H580\n" → emit humi(58.0)  │   ├── captureRequested → onCapturePhoto
│   ├── "K1\n"   → emit freeze()    │   └── alarmReceived → 状态栏变红
│   ├── "K2\n"   → emit capture()   │
│   └── "A2\n"   → emit alarm(2)    ├── LED 亮度滑块变化
├── write("L75\n")                   │   → sendCommand("L75")
└── 循环                             └── 状态栏: "温度: 25.5°C / 湿度: 58.0%"
```

## 三、新增文件

### 3.1 src/comm/rpmsglistener.h

RPMsg 监听线程声明。继承 QThread，通过信号槽与主线程通信。

关键信号：
- `temperatureUpdated(double)` — 温度更新
- `humidityUpdated(double)` — 湿度更新
- `freezeRequested()` — KEY1 按下
- `captureRequested()` — KEY2 按下
- `alarmReceived(int)` — 温度报警
- `connectionChanged(bool)` — 连接状态变化

### 3.2 src/comm/rpmsglistener.cpp

实现：
- `run()` — 打开 `/dev/ttyRPMSG0`，用 `select()` 轮询读取（100ms 超时），逐行解析协议
- `sendCommand()` — 向 M4 发送指令（如 `L75\n`）
- `parseLine()` — 解析 M4 上报的数据（T/H/K/A 前缀）

## 五、遇到的问题

### 问题 1：TIM2 导致 LCD 背光熄灭（问题 #22）

LCD 背光由 TIM2_CH1(PA15) PWM 控制。M4 初始化 TIM2 时破坏了背光 PWM。

**解决**：M4 移除 TIM2，LED 改用 TIM3 中断中的软件 PWM。

### 问题 2：I2C1 与触摸屏冲突（问题 #24）

触摸屏和 SHT30 共享 I2C1(PF14/PF15)。M4 初始化 I2C1 破坏触摸屏通信。

**解决**：暂时禁用 I2C1，温度功能待确认空闲引脚后用软件 I2C 恢复。

### 问题 3：冻结功能在三线程模式下失效

三线程管线的 displayTimer 回调中没有填充 `m_frameBuffer`，导致冻结时 buffer 为空。

**解决**：在 displayTimer 回调中添加 `m_frameBuffer.push_back(frame)`，冻结时跳过帧更新。

### 问题 4：remoteproc stop/start 后 RPMsg 不工作

第二次 stop/start M4 后 RPMsg 数据通道断开。

**解决**：操作规范 — 每次更新 M4 固件后重启开发板。
