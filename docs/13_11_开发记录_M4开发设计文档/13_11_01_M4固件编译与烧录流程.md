## 二、完整流程

### 步骤 1：编译（虚拟机）

```bash
# 方法 A：CubeIDE 图形界面
# 打开 CubeIDE → 选中 CM4 工程 → Project → Build Project (Ctrl+B)

# 方法 B：命令行 headless 编译
/opt/st/stm32cubeide_1.19.0/headless-build.sh \
  -data /tmp/m4_workspace \
  -import /mnt/hgfs/VM_Share/project/EndoscopeM4/CM4 \
  -build "EndoscopeM4_CM4/Debug"
```

编译成功后输出：
```
Finished building target: EndoscopeM4_CM4.elf
   text    data     bss     dec     hex  filename
  22772      12    1788   24572    5ffc  EndoscopeM4_CM4.elf
0 errors, 0 warnings.
```

### 步骤 2：传输到 NFS 共享目录（虚拟机）

```bash
cp /mnt/hgfs/VM_Share/project/EndoscopeM4/CM4/Debug/EndoscopeM4_CM4.elf /home/chris/nfs_share/
```

### 步骤 3：复制到 firmware 目录（开发板）

```bash
cp /mnt/nfs/EndoscopeM4_CM4.elf /lib/firmware/
```

### 步骤 4：加载并启动 M4（开发板）

```bash
# 指定固件文件名
echo EndoscopeM4_CM4.elf > /sys/class/remoteproc/remoteproc0/firmware

# 启动 M4
echo start > /sys/class/remoteproc/remoteproc0/state

# 确认运行状态
cat /sys/class/remoteproc/remoteproc0/state
# 输出: running
```

### 步骤 5：停止 M4（开发板）

```bash
echo stop > /sys/class/remoteproc/remoteproc0/state

# 确认已停止
cat /sys/class/remoteproc/remoteproc0/state
# 输出: offline
```

> **注意**：停止后 PB10 可能保持最后电平状态导致 LED 常亮，手动关闭：
> ```bash
> echo 26 > /sys/class/gpio/export 2>/dev/null
> echo out > /sys/class/gpio/gpio26/direction
> echo 0 > /sys/class/gpio/gpio26/value
> echo 26 > /sys/class/gpio/unexport
> ```
> 后续正式代码会处理停止信号，自动归零 PWM。

## 三、一键脚本

### 虚拟机侧：编译 + 传输

```bash
#!/bin/bash
# build_and_deploy.sh — 在虚拟机上执行

echo "=== 编译 M4 固件 ==="
/opt/st/stm32cubeide_1.19.0/headless-build.sh \
  -data /tmp/m4_workspace \
  -import /mnt/hgfs/VM_Share/project/EndoscopeM4/CM4 \
  -build "EndoscopeM4_CM4/Debug" 2>&1 | tail -5

if [ $? -eq 0 ]; then
    echo "=== 复制到 NFS 共享目录 ==="
    cp /mnt/hgfs/VM_Share/project/EndoscopeM4/CM4/Debug/EndoscopeM4_CM4.elf /home/chris/nfs_share/
    echo "完成！在开发板上执行 m4_run.sh 加载固件"
else
    echo "编译失败！"
fi
```

### 开发板侧：加载 + 运行

```bash
#!/bin/bash
# m4_run.sh — 在开发板上执行

# 停止旧固件
echo stop > /sys/class/remoteproc/remoteproc0/state 2>/dev/null

# 复制新固件
cp /mnt/nfs/EndoscopeM4_CM4.elf /lib/firmware/

# 加载并启动
echo EndoscopeM4_CM4.elf > /sys/class/remoteproc/remoteproc0/firmware
echo start > /sys/class/remoteproc/remoteproc0/state

echo "M4 状态: $(cat /sys/class/remoteproc/remoteproc0/state)"
```

### 开发板侧：停止

```bash
#!/bin/bash
# m4_stop.sh — 在开发板上执行

echo stop > /sys/class/remoteproc/remoteproc0/state 2>/dev/null

# 关闭 LED（清除 PB10 残留电平）
echo 26 > /sys/class/gpio/export 2>/dev/null
echo out > /sys/class/gpio/gpio26/direction
echo 0 > /sys/class/gpio/gpio26/value
echo 26 > /sys/class/gpio/unexport 2>/dev/null

echo "M4 已停止, LED 已关闭"
```

## 五、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `echo start` 报错 | 固件文件不存在 | 确认 `/lib/firmware/EndoscopeM4_CM4.elf` 存在 |
| `echo stop` 报 Invalid argument | 已经是 offline | 正常，忽略 |
| `warning: remote FW shutdown without ack` | M4 未实现 RPMsg 通信 | 正常，后续加 OpenAMP 后消失 |
| 停止后 LED 常亮 | PB10 保持最后电平 | 用 GPIO sysfs 手动拉低，或用 m4_stop.sh |
| NFS 挂载失败 | WiFi 未连接或 IP 变化 | `ifconfig` 确认双方 IP |
