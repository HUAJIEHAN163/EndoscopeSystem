# 13_3 开发记录：硬件参数区与白平衡卸载

## 一、背景

### 当前问题

调试模式的右侧参数面板已经比较拥挤，所有控件（白平衡、CLAHE、去雾、锐化、降噪、边缘检测、阈值分割）都堆在一起，没有区分硬件和软件的概念。

同时，软件白平衡（`applyWhiteBalance`）和 OV5640 硬件自动白平衡存在**重复处理**：

```
OV5640 Sensor 内部 ISP
    ↓ 已做自动白平衡
DCMI 采集
    ↓
软件 applyWhiteBalance   ← 重复处理，浪费 9ms/帧
    ↓
CLAHE、去雾、锐化...
```

### 为什么要做这个改动

1. **性能**：软件白平衡每帧耗时 ~9ms，关掉后 FPS 提升约 20-25%，同时降低 overrun 风险

> → FPS 问题的完整跟踪见 `16_FPS性能问题总跟踪.md`
2. **正确性**：OV5640 已经做过白平衡，软件再做一遍是重复计算，且两者算法不同可能产生色偏叠加
3. **界面**：硬件参数和软件参数混在一起，不清晰；分区后逻辑更清楚，也为后续添加曝光、增益控制留出位置

## 二、白平衡为什么应该常开

内窥镜场景下，白平衡必须始终开启，原因如下：

**1. 光源色温不稳定**

内窥镜使用 LED 冷光源，虽然色温相对固定，但：
- 进入不同腔道（胃、肠、支气管），反射光的色调不同
- 距离黏膜远近变化时，画面亮度和色偏会变化
- 白平衡能自动补偿这些变化，保持色彩稳定

**2. 医生依赖色彩判断病灶**

内窥镜诊断中，颜色是重要的判断依据：
- 发红区域 → 可能是炎症、出血
- 发白区域 → 可能是溃疡、坏死
- 发黄区域 → 可能是胆汁反流

色偏会直接干扰医生判断，白平衡保证黏膜颜色尽量还原真实。

**3. 硬件白平衡优于软件白平衡**

OV5640 的自动白平衡（AWB）在 Sensor 内部完成：
- 在原始 RAW 数据阶段处理，精度更高
- 不占用 CPU，零额外开销
- 响应速度更快（逐帧实时调整）

软件白平衡（灰度世界法）是一种简化算法，假设图像整体均值为灰色，在内窥镜场景（大面积红色黏膜）下假设不成立，效果反而不如硬件 AWB。

**结论：白平衡常开，但由 OV5640 硬件完成，软件不再重复处理。**

## 四、实施步骤

### Step 1：v4l2capture 新增硬件控制方法

在 `v4l2capture.h` 中声明：

```cpp
// 硬件参数控制（通过 V4L2 ioctl 直接控制 OV5640）
void setAutoWhiteBalance(bool enable);
void setAutoExposure(bool enable);
void setHorizontalFlip(bool enable);
void setVerticalFlip(bool enable);
```

在 `v4l2capture.cpp` 中实现，并在 `open()` 末尾默认开启硬件白平衡：

```cpp
// open() 末尾添加
setAutoWhiteBalance(true);   // 默认开启硬件白平衡，替代软件 applyWhiteBalance
```

### Step 2：mainwindow 新增硬件参数分组框

在 `setupUi()` 的调试面板中，在内窥镜算法分组框之前插入硬件参数分组框：

```cpp
QGroupBox *hwGroup = new QGroupBox("硬件参数 (OV5640)");
// 添加：自动白平衡、自动曝光、水平翻转、垂直翻转
```

硬件参数控件的信号槽直接调用 v4l2capture 的控制方法，不走 processFrame。

### Step 3：删除软件白平衡

- 删除 `m_chkWhiteBalance` 复选框
- `processFrame` 中不再读取白平衡状态
- `Config::whiteBalance` 默认值改为 `false`

### Step 4：虚拟机兼容处理

硬件参数区只在开发板（有真实摄像头）时有意义。虚拟机使用 FileCapture 时，硬件参数控件应置灰：

```cpp
// setupVideoSource() 完成后
bool hasHardware = (dynamic_cast<V4l2Capture*>(m_source) != nullptr);
m_hwGroup->setEnabled(hasHardware);
```

2. 硬件白平衡常开，画质有保障
3. 硬件和软件参数分区显示，界面更清晰
4. 为后续硬件参数控制留好框架（曝光、增益、翻转都已接好）

### 7.4 mainwindow.cpp — setupVideoSource()：硬件参数置灰 + 信号槽连接

在 `m_source->start()` 之后新增：

```cpp
    // 硬件参数区：只有 V4L2 摄像头时才可用
#ifdef ENABLE_V4L2
    bool hasHardware = (dynamic_cast<V4l2Capture*>(m_source) != nullptr);
#else
    bool hasHardware = false;
#endif
    m_hwGroup->setEnabled(hasHardware);
    if (!hasHardware)
        m_hwGroup->setToolTip("仅开发板摄像头可用");

    // 开发板上默认开启硬件白平衡和自动曝光
#ifdef ENABLE_V4L2
    if (hasHardware) {
        auto *v4l2 = dynamic_cast<V4l2Capture*>(m_source);
        v4l2->setAutoWhiteBalance(true);
        v4l2->setAutoExposure(true);

        // 连接硬件参数控件信号槽
        connect(m_chkAutoWhiteBalance, &QCheckBox::toggled, [v4l2](bool checked) {
            v4l2->setAutoWhiteBalance(checked);
        });
        connect(m_chkAutoExposure, &QCheckBox::toggled, [v4l2](bool checked) {
            v4l2->setAutoExposure(checked);
        });
        connect(m_chkHFlip, &QCheckBox::toggled, [v4l2](bool checked) {
            v4l2->setHFlip(checked);
        });
        connect(m_chkVFlip, &QCheckBox::toggled, [v4l2](bool checked) {
            v4l2->setVFlip(checked);
        });
    }
#endif
```
