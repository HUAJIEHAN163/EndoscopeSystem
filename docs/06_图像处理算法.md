# 06 — 图像处理算法

## 6.1 算法总览

共 8 个算法，分为三类:

| 类别 | 算法 | 内窥镜场景 | 来源 |
|------|------|-----------|------|
| 内窥镜专用 | 白平衡 (灰度世界法) | LED 光源色偏 | 新增 |
| 内窥镜专用 | CLAHE 自适应直方图均衡化 | 体腔内光照不均 | 新增 |
| 内窥镜专用 | 畸变校正 (remap) | 广角镜头桶形畸变 | 新增 |
| 内窥镜专用 | 去雾 (暗通道先验) | 镜头起雾 | 新增 |
| 通用增强 | USM 锐化 | 增强组织纹理 | 改进自 VideoMonitor |
| 通用增强 | 双边滤波降噪 | 低光噪声 | 新增 |
| 经典保留 | Canny 边缘检测 | 辅助观察边界 | 保留自 VideoMonitor |
| 经典保留 | 阈值分割 | 辅助分析 | 保留自 VideoMonitor |

### 处理顺序 (固定)

```
白平衡 → CLAHE → 畸变校正 → 去雾 → 锐化 → 降噪 → 边缘检测 → 阈值分割
```

顺序设计理由:
- 白平衡先校正色偏，后续算法基于正确颜色工作
- CLAHE 在去畸变前做，避免畸变校正引入的黑边影响均衡化
- 去雾在锐化前，先恢复清晰度再增强细节
- 降噪在锐化后，避免锐化放大噪声
- 边缘检测/阈值分割是分析类算法，放最后

## 6.2 内窥镜专用算法

### 算法 1: CLAHE 自适应直方图均衡化

**原理**: 将图像分成小块 (tile)，对每块分别做直方图均衡化，
用 clipLimit 限制对比度放大倍数，避免噪声被放大。

**内窥镜意义**: 体腔内 LED 光源照射不均匀，近处过亮远处过暗。
CLAHE 能同时增强亮区和暗区的细节，是内窥镜最常用的增强算法。

**实现要点**:
- 在 Lab 色彩空间只处理 L 通道 (亮度)，保留原始色彩
- clipLimit 默认 3.0，可通过滑块调节 (1.0-8.0)
- tileSize 固定 8x8

**参数**:
| 参数 | 范围 | 默认 | 影响 |
|------|------|------|------|
| clipLimit | 1.0-8.0 | 3.0 | 越大对比度越强，但噪声也越明显 |

### 算法 2: 畸变校正

**原理**: 根据相机标定得到的内参矩阵和畸变系数，
预计算像素映射表 (initUndistortRectifyMap)，运行时用 remap 实时校正。

**内窥镜意义**: 内窥镜使用广角/鱼眼镜头，产生严重桶形畸变，
影响医生对组织形态和大小的判断。这是内窥镜的核心功能。

**实现要点**:
- 映射表只需计算一次 (程序启动时)，后续 remap 很快 (3-5ms)
- 需要标定文件 `camera_calib.yml`，没有则此功能不可用
- 标定需要用内窥镜拍摄棋盘格照片 (至少 15 张不同角度)

### 算法 3: 去雾 (暗通道先验)

**原理**: 何恺明的暗通道先验理论 — 在无雾图像的非天空区域，
至少有一个颜色通道的值接近 0。利用这个先验估计雾浓度并去除。

**内窥镜意义**: 体腔内温差导致镜头起雾，画面变白变模糊。

**实现要点**:
- 计算量较大 (20-40ms @640x480)，不建议与其他重算法同时开启
- omega 参数控制去雾强度 (0.95 = 去除 95% 的雾)
- 透射率下限 0.1，避免除零产生伪影

**参数**:
| 参数 | 范围 | 默认 | 影响 |
|------|------|------|------|
| omega | 0.5-1.0 | 0.95 | 越大去雾越强 |
| radius | 3-15 | 7 | 暗通道窗口大小 |

### 算法 4: 白平衡 (灰度世界法)

**原理**: 假设场景中所有颜色的平均值应该是灰色，
据此计算 B/G/R 三通道的增益系数进行校正。

**内窥镜意义**: LED 光源色温偏差导致画面偏蓝或偏黄。

**实现要点**:
- 计算量极小 (2-3ms)，可以常开
- 分母加 1e-6 防止除零

## 6.3 通用增强算法

### 算法 5: USM 锐化

**原理**: 用高斯模糊得到低频分量，原图减去低频得到高频细节，
再将高频细节按比例叠加回原图: `dst = src * (1+amount) - blur * amount`

**内窥镜意义**: 增强组织纹理和边缘，帮助观察细微病变。

**与 VideoMonitor 的区别**: 原项目直接用 `addWeighted(src, 1.5, blur, -0.5, 0)`，
本项目 amount 参数可通过滑块调节。

**参数**:
| 参数 | 范围 | 默认 | 影响 |
|------|------|------|------|
| amount | 0.5-3.0 | 1.5 | 越大锐化越强，过大会产生振铃 |

### 算法 6: 双边滤波降噪

**原理**: 在空间域和像素值域同时加权滤波，
既能平滑噪声，又能保留边缘 (不像高斯滤波会模糊边缘)。

**内窥镜意义**: 低光环境 + 小尺寸传感器导致噪声大，但不能模糊组织边缘。

**实现要点**:
- d=5 (不要设太大，d=9 速度慢 3 倍)
- 计算量较大 (15-30ms)

## 6.4 保留的经典算法

### 算法 7: Canny 边缘检测

保留自 VideoMonitor，用于辅助观察组织边界。
输出为灰度边缘图 (转回 BGR 以保持管线一致)。

### 算法 8: 阈值分割

保留自 VideoMonitor，用于辅助分析。
阈值可通过滑块调节 (0-255)。

## 6.5 性能预估 (STM32MP157, 640x480)

| 算法 | 预估耗时 | 说明 |
|------|----------|------|
| 白平衡 | 2-3ms | 通道均值计算，极快 |
| CLAHE | 5-10ms | 只处理 L 通道 |
| 畸变校正 (remap) | 3-5ms | 预计算映射表后很快 |
| 去雾 | 20-40ms | 计算量大 |
| USM 锐化 | 3-5ms | 一次高斯模糊 + 加权 |
| 双边滤波 | 15-30ms | d=5 |
| 边缘检测 | 3-5ms | |
| 阈值分割 | 1-2ms | |

**建议**: 默认开启白平衡 + CLAHE，其他按需。不要同时开启去雾 + 降噪 (总耗时可能超过 33ms/帧)。
