## 5.2 控制面板分组

### 内窥镜算法组

| 控件 | 类型 | 功能 |
|------|------|------|
| 白平衡校正 | QCheckBox | 灰度世界法白平衡 |
| CLAHE增强 | QCheckBox | 自适应直方图均衡化 |
| CLAHE 滑块 | QSlider (1.0-8.0) | clipLimit 参数 |
| 畸变校正 | QCheckBox | 镜头去畸变 (需标定文件) |
| 去雾 | QCheckBox | 暗通道先验去雾 |

### 通用算法组

| 控件 | 类型 | 功能 |
|------|------|------|
| 锐化 | QCheckBox | USM 锐化 |
| 锐化滑块 | QSlider (0.5-3.0) | amount 参数 |
| 降噪 | QCheckBox | 双边滤波 |
| 边缘检测 | QCheckBox | Canny (保留自 VideoMonitor) |
| 阈值分割 | QCheckBox + QSlider (0-255) | 二值化 (保留自 VideoMonitor) |

### 操作组

| 按钮 | 快捷键 | 功能 |
|------|--------|------|
| 拍照 | C | 保存当前帧为 JPEG |
| 录像 | R | 开始/停止录像 (AVI MJPG) |
| 冻结 | Space | 暂停/继续画面更新 |
| 旋转 | T | 90°/180°/270°/0° 循环 |
| 退出 | ESC | 关闭程序 |

## 5.3 OSD (On-Screen Display)

在 paintEvent 中叠加:

- **时间戳**: 左上角，黄色，`yyyy-MM-dd hh:mm:ss`
- **录像指示**: 红色圆点 + "REC" 文字 (仅录像时显示)
- **FPS**: 右侧控制面板中显示

## 5.4 视频源初始化逻辑

```
启动 → 检测 /dev/video0 是否存在
  ├─ 存在 → 创建 V4l2Capture → open()
  │   ├─ 成功 → 使用 V4L2 采集
  │   └─ 失败 → 降级为文件模式
  └─ 不存在 → 读取 config/endoscope.conf 中的 file/path
      ├─ 文件存在 → 创建 FileCapture → open()
      └─ 文件不存在 → 显示错误: "无视频源"
```

## 5.5 帧处理流程

```cpp
onFrameReady(QImage image):
  if (冻结中) return;
  frameCount++;

  // 1. 图像处理
  读取所有 checkbox/slider 状态 → 构建 Config
  if (任何算法启用):
    QImage→Mat → ImageProcessor::process(config) → Mat→QImage

  // 2. 旋转
  if (rotateAngle != 0):
    QTransform.rotate → transformed

  // 3. 录像
  if (录像中):
    QImage→Mat → VideoWriter.write

  // 4. 缩放显示
  scaled(videoRect, KeepAspectRatio) → m_displayImage
  update() → 触发 paintEvent
```

## 5.6 与原 VideoMonitor Client 的区别

| 原 VideoMonitor Client | 本项目 |
|-----------------------|--------|
| 多窗口管理 (OneVideo 多实例) | 单窗口 |
| TCP 接收图片 (RecvThread) | 本地采集 (VideoSource) |
| 登录系统 (SQLite) | 无 (单机设备) |
| 3 个算法 checkbox | 8 个算法 + 参数滑块 |
| 录像功能未完成 (空函数) | 完整实现 |
| 无冻结/旋转 | 完整实现 |
| 无 OSD | 时间戳 + 录像指示 |
| 无 FPS 显示 | 实时 FPS |
| 无配置文件 | endoscope.conf |
