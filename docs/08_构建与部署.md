# 08 — 构建与部署

## 8.1 构建系统

使用 CMake，支持两种构建模式:

| 模式 | 用途 | 编译器 |
|------|------|--------|
| 本地构建 | 虚拟机开发调试 | gcc/g++ (x86) |
| 交叉编译 | 部署到开发板 | arm-linux-gnueabihf-g++ |

---

## 8.2 本地构建 (虚拟机)

```bash
cd /mnt/hgfs/VM_Share/project/EndoscopeSystem
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Debug
make -j$(nproc)

# 运行
./endoscope
```

或使用脚本:

```bash
bash scripts/build_local.sh
```

### CMakeLists.txt 关键配置

```cmake
# Qt5 自动 MOC 处理 (Q_OBJECT 宏)
set(CMAKE_AUTOMOC ON)

# V4L2 条件编译
option(ENABLE_V4L2 "Enable V4L2" ON)
if(ENABLE_V4L2 AND CMAKE_SYSTEM_NAME STREQUAL "Linux")
    add_definitions(-DENABLE_V4L2)
endif()

# 依赖
find_package(Qt5 REQUIRED COMPONENTS Core Gui Widgets)
find_package(OpenCV REQUIRED COMPONENTS core imgproc imgcodecs videoio)
```

## 8.3 交叉编译 (ARM, Phase 4)

### 前提条件 (均已完成, 见 01_开发环境搭建.md)

- Linaro gcc 7.5.0: `/opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf/`
- ARM 版 Qt5 5.11.3: `/opt/qt-everywhere-src-5.11.3/`
- ARM 版 OpenCV 3.4.16: `/opt/opencv_arm/`

> ⚠️ **必须用 Linaro gcc 7.5.0**, 不能用 apt 的交叉编译器 (glibc 2.39 与开发板 2.28 不兼容)。

### 编译命令

```bash
export PATH=/opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf/bin:$PATH

mkdir build_arm && cd build_arm
cmake .. \
    -DCMAKE_TOOLCHAIN_FILE=../scripts/arm_toolchain.cmake \
    -DCMAKE_BUILD_TYPE=Release \
    -DQt5_DIR=/opt/qt-everywhere-src-5.11.3/lib/cmake/Qt5 \
    -DOpenCV_DIR=/opt/opencv_arm/lib/cmake/opencv4
make -j$(nproc)

# 验证是 ARM 可执行文件
file endoscope
# 期望: ELF 32-bit LSB executable, ARM, ...
```

### 工具链文件 (scripts/arm_toolchain.cmake)

```cmake
set(CMAKE_SYSTEM_NAME Linux)
set(CMAKE_SYSTEM_PROCESSOR arm)

# 必须使用 Linaro gcc 7.5.0 (glibc 2.25, 兼容开发板 glibc 2.28)
# apt 的交叉编译器 glibc 2.39 不兼容!
set(TOOLCHAIN_ROOT /opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf)
set(CMAKE_C_COMPILER   ${TOOLCHAIN_ROOT}/bin/arm-linux-gnueabihf-gcc)
set(CMAKE_CXX_COMPILER ${TOOLCHAIN_ROOT}/bin/arm-linux-gnueabihf-g++)

# Cortex-A7 NEON 优化
set(CMAKE_C_FLAGS   "-march=armv7-a -mfpu=neon-vfpv4 -mfloat-abi=hard")
set(CMAKE_CXX_FLAGS "-march=armv7-a -mfpu=neon-vfpv4 -mfloat-abi=hard")
```

## 8.4 部署到开发板 (Phase 4)

```bash
# 1. 拷贝可执行文件
scp build_arm/endoscope root@192.168.0.43:/opt/endoscope/

# 2. 拷贝配置文件
scp config/endoscope.conf root@192.168.0.43:/opt/endoscope/config/

# 3. OpenCV 库已部署到开发板 (17个 .so, 见 01_开发环境搭建.md)

# 4. 在开发板上运行
ssh debian@192.168.0.43   # 密码: temppwd
su -                       # 密码: root
export QT_QPA_PLATFORM=linuxfb:fb=/dev/fb0
/opt/endoscope/endoscope
```

## 8.5 配置文件

```ini
# config/endoscope.conf

[camera]
device=/dev/video0
width=640
height=480
fps=30
buffers=4

[file]
path=test_data/test.mp4
```

虚拟机和开发板使用同一份代码，只需修改配置文件中的路径。

## 8.6 开机自启动 (可选, Phase 4)

```bash
# systemd service
cat > /etc/systemd/system/endoscope.service << 'EOF'
[Unit]
Description=Endoscope System
After=multi-user.target

[Service]
Type=simple
ExecStart=/opt/endoscope/endoscope
Environment=QT_QPA_PLATFORM=linuxfb:fb=/dev/fb0
Restart=on-failure

[Install]
WantedBy=multi-user.target
EOF

systemctl enable endoscope
```
