## 一、背景

### 当前问题

contrast（对比度）和saturation（饱和度）没有被使用

参数	当前默认值	作用	内窥镜场景是否需要
contrast（对比度）	0	增大明暗差异，看清细节	有用，暗区细节更清晰
saturation（饱和度）	64	颜色鲜艳程度	有用，适当提高能让病灶颜色更明显
hue（色调）	0	整体色彩偏移	一般不调，调错会导致色彩失真

这些参数和 CLAHE 的区别是：它们在硬件端完成，不占 CPU。CLAHE 是软件算法每帧要算 28ms，而对比度/饱和度是 OV5640 内部寄存器直接调整，零开销。

### 建议：

可以在硬件参数区的折叠面板里加上对比度和饱和度的滑块，让工程师在调试模式下调参，找到最佳值后固定下来。

## 三、链路解释

### 1 完整链路：
用户拖动饱和度滑块
    ↓
Qt 发出 valueChanged(128) 信号
    ↓
lambda 被调用：v4l2->setSaturation(128)
    ↓
setSaturation → setControl(V4L2_CID_SATURATION, 128)
    ↓
ioctl(m_fd, VIDIOC_S_CTRL, &ctrl)    ← 用户空间 → 内核空间
    ↓
Linux 内核 V4L2 子系统
    ↓
OV5640 驱动（drivers/media/i2c/ov5640.c）
    ↓
I2C 总线写入 OV5640 寄存器              ← 内核 → 硬件
    ↓
OV5640 Sensor 内部 ISP 更新饱和度参数
    ↓
下一帧图像的饱和度就变了

### 2 auto *v4l2 = dynamic_cast<V4l2Capture*>(m_source);
m_source 是 VideoSource* 类型（基类指针），它实际指向的可能是 V4l2Capture 或 FileCapture。dynamic_cast 的作用是把基类指针安全地转换为子类指针：

VideoSource *m_source;  // 基类指针，不知道具体是哪个子类

// dynamic_cast 尝试转换：
// 如果 m_source 实际指向 V4l2Capture → 转换成功，返回 V4l2Capture*
// 如果 m_source 实际指向 FileCapture → 转换失败，返回 nullptr
auto *v4l2 = dynamic_cast<V4l2Capture*>(m_source);

转换成功后，v4l2 就能调用 V4l2Capture 独有的方法（setSaturation 等），而 m_source 不能调用这些方法（基类没有声明）。

### 3 [v4l2](int value) { v4l2->setSaturation(value); 
这是 lambda 表达式（匿名函数），拆开看：
[v4l2]          // 捕获列表：把外部变量 v4l2 指针拷贝进来
(int value)     // 参数：滑块的当前值，由 Qt 信号传入
{
    v4l2->setSaturation(value);  // 函数体：调用 v4l2 的方法
}
当用户拖动滑块时，Qt 自动调用这个 lambda，把滑块的值传给 value

### 4 v4l2->setSaturation(value) 怎么控制硬件
// v4l2capture.cpp
bool V4l2Capture::setSaturation(int value) {
    return setControl(V4L2_CID_SATURATION, value);
}

bool V4l2Capture::setControl(uint32_t id, int value) {
    struct v4l2_control ctrl;
    ctrl.id = id;        // 参数 ID，如 V4L2_CID_SATURATION
    ctrl.value = value;  // 要设置的值，如 128
    return xioctl(m_fd, VIDIOC_S_CTRL, &ctrl) == 0;
}
xioctl 最终调用 Linux 的 ioctl 系统调用，这是用户程序和内核驱动通信的接口。
