## 二、整体架构

### 2.1 三线程 + 两个队列

```
采集线程 (V4l2Capture::run)
  │
  │ push(std::move(bgr))     ← cv::Mat BGR 640×480
  ▼
captureQueue (FrameQueue<4>)   ← 无锁环形队列，4 个槽位
  │
  │ pop(frame)
  ▼
处理线程 (ProcessThread::run)
  │
  │ push(std::move(result))   ← cv::Mat BGR 460×360（已 resize）
  ▼
displayQueue (FrameQueue<4>)   ← 无锁环形队列，4 个槽位
  │
  │ latest(mat)
  ▼
主线程 (displayTimer 16ms 回调)
  │
  ▼
paintEvent → 屏幕显示
```

### 2.2 源文件与职责

```
src/
├── main.cpp                         # 程序入口，创建 QApplication 和 MainWindow
├── capture/
│   ├── videosource.h                # 视频源抽象基类（定义接口）
│   ├── v4l2capture.h/cpp            # V4L2 采集实现（开发板）— 采集线程
│   └── filecapture.h/cpp            # 视频文件输入（虚拟机测试）— 采集线程
├── processing/
│   ├── imageprocessor.h/cpp         # 图像处理算法集合（8 个算法）
│   ├── processthread.h              # 处理线程（从 captureQueue 取帧 → 算法 → displayQueue）
│   └── neon_accel.h/cpp             # NEON 加速模块（当前未使用，P5 尝试后保留）
├── ui/
│   ├── mainwindow.h/cpp             # 主窗口（UI + 显示 + 交互 + 三线程管理）
│   └── presetmanager.h/cpp          # 预设文件管理（JSON 读写）
└── utils/
    ├── framequeue.h                 # 无锁环形队列模板（线程间传递 cv::Mat）
    └── imageconvert.h/cpp           # QImage ↔ cv::Mat 转换（图像编辑页使用）
```

## 三、启动流程

### 3.1 调用链

```
main()
  ├── QApplication app(argc, argv)        // Qt 事件循环
  ├── MainWindow window                   // 构造函数触发所有初始化
  │     ├── setupUi()                     // 创建界面控件
  │     ├── loadPresets()                 // 加载 config/presets/*.json
  │     ├── setupVideoSource()            // 初始化视频源和三线程管线
  │     │     ├── V4l2Capture::open()     // 或 FileCapture::open()
  │     │     │     ├── queryCapability() // VIDIOC_QUERYCAP
  │     │     │     ├── enumFormats()     // VIDIOC_ENUM_FMT
  │     │     │     ├── tryAndSetFormat() // VIDIOC_TRY_FMT + VIDIOC_S_FMT
  │     │     │     ├── setFrameRate()    // VIDIOC_S_PARM
  │     │     │     └── requestBuffers()  // VIDIOC_REQBUFS + mmap + VIDIOC_QBUF
  │     │     ├── m_source->m_captureQueue = &m_captureQueue
  │     │     ├── new ProcessThread(&captureQueue, &displayQueue)
  │     │     ├── new QTimer (displayTimer, 16ms)
  │     │     ├── m_source->start()       // 启动采集线程（QThread::start → run）
  │     │     └── m_processThread->start() // 启动处理线程
  │     ├── 加载 camera_calib.yml         // 畸变校正映射表（可选）
  │     └── CLAHE 预热                    // 8×8 dummy Mat，消除首次 3s 卡顿
  ├── window.show()                       // 显示窗口
  └── app.exec()                          // 进入 Qt 事件循环（阻塞直到退出）
```
```
    // 启动采集线程和处理线程
    m_source->start();
    m_processThread->start();
    tart() 是 QThread 的方法，它会创建一个新的操作系统线程，然后在新线程中调用 run()：
    m_source->start()
  │ QThread 内部：创建新线程
  └── 新线程中自动调用 → V4l2Capture::run()
                          │
                          while (!m_quit) {
                              grabFrame()
                              cvtColor()
                              push()
                          }

m_processThread->start()
  │ QThread 内部：创建新线程
  └── 新线程中自动调用 → ProcessThread::run()
                          │
                          while (!m_quit) {
                              pop()
                              process/resize()
                              push()
                          }

start() 调用后立即返回，不会等 run() 执行完。所以 setupVideoSource() 中这两行执行完后，三个线程就同时在跑了：
m_source->start();          // 采集线程开始跑，start() 立即返回
m_processThread->start();   // 处理线程开始跑，start() 立即返回
// 继续执行后面的代码...    // 主线程继续

之后的状态：
start() → run()，直接去看 run() （start() 是 QThread 提供的，你永远不需要改它。你只需要写 run() 里面的逻辑。）
每个线程类只需要关注它的 run() 方法：
类	run() 在哪里	做什么
V4l2Capture	v4l2capture.cpp	grabFrame → cvtColor → push 循环
FileCapture	filecapture.cpp	VideoCapture.read → push 循环
ProcessThread	processthread.h	pop → process/resize → push 循环

时间线：
─────────────────────────────────────────────────────→

采集线程：  grabFrame→cvtColor→push → grabFrame→cvtColor→push → ...
处理线程：  (等队列有数据) → pop→resize→push → pop→resize→push → ...
主线程：    setupVideoSource()返回 → 进入 app.exec() 事件循环 → 每16ms displayTimer回调 → ...

```
### 3.2 启动后三个线程的状态

```
启动完成后，三个线程同时运行：

采集线程 (V4l2Capture::run):
  while (!m_quit) {
      grabFrame → cvtColor(YUYV→BGR) → returnBuffer → push(move) → 循环
  }

处理线程 (ProcessThread::run):
  while (!m_quit) {
      pop(frame) → process/resize → push(move) → 循环
  }

主线程 (Qt 事件循环):
  每 16ms: displayTimer 回调 → latest(mat) → BGR→RGB → QImage → update → paintEvent
  随时:    按钮点击 → onCapturePhoto / onToggleRecord / ...
  每 1s:   fpsTimer → updateFps
```

## 四、一帧图像的完整生命周期

以不开算法为例，追踪一帧图像从摄像头到屏幕的完整路径：

### 4.1 阶段 1：硬件采集（V4L2 驱动层）

```
OV5640 Sensor
  │ 硬件 ISP：自动白平衡、自动曝光、对比度、饱和度
  ▼
DCMI (Digital Camera Memory Interface)
  │ DMA 传输到内核 buffer（YUYV 格式，640×480×2 = 614KB）
  ▼
V4L2 驱动 (stm32-dcmi)
  │ buffer 就绪，select() 返回
  ▼
用户空间
```

数据格式：**YUYV 4:2:2 packed**
- 每 4 字节编码 2 个像素：Y0 U Y1 V
- 每个像素有独立的 Y（亮度），相邻两像素共享 U/V（色度）
- 总大小：640 × 480 × 2 = 614,400 字节

### 4.2 阶段 2：采集线程处理

```
文件：v4l2capture.cpp — run()

grabFrame(&data, &size, &bufIndex)
  │ select() 等待数据就绪
  │ VIDIOC_DQBUF 取出 buffer
  │ data = mmap 地址（指向 YUYV 数据）
  ▼
cv::Mat yuyv(480, 640, CV_8UC2, data)    // 包装为 Mat，不拷贝数据
  │                                        // CV_8UC2 = 8bit × 2通道 = YUYV
  ▼
cv::cvtColor(yuyv, bgr, COLOR_YUV2BGR_YUYV)
  │ YUYV → BGR 颜色转换（OpenCV 内置 NEON 优化）
  │ 输入：614KB YUYV → 输出：921KB BGR（640×480×3）
  │ 耗时：17-55ms（平均 ~33ms）
  ▼
returnBuffer(bufIndex)                    // 归还 V4L2 buffer（必须在 cvtColor 之后）
  ▼
m_captureQueue->push(std::move(bgr))     // 零拷贝转移到队列（P16）
  │ bgr 的数据指针转移给队列槽位
  │ bgr 变为空 Mat
  │ 耗时：~0ms
  ▼
数据现在在 captureQueue 中
格式：cv::Mat BGR 640×480 CV_8UC3（921KB）
```

### 4.3 阶段 3：处理线程处理

```
文件：processthread.h — run()

m_captureQueue->pop(frame)               // 从队列取一帧
  │ frame = 浅拷贝（共享数据，引用计数 +1）
  ▼
不开算法时：
  cv::resize(frame, result, Size(460, 360))
  │ 640×480 → 460×360 双线性插值
  │ 输出新 Mat（497KB）
  │ 耗时：22-59ms（CPU 调度波动）
  ▼
开 CLAHE 时：
  ImageProcessor::process(frame, dst, config, map1, map2)
  │ 内部流程：
  │   resize(640→320)           ~2ms
  │   applyCLAHE(320×240)       ~28ms   BGR→Lab→均衡L→Lab→BGR
  │   applyDehaze(320×240)      ~87ms   （如果开启）
  │   dst = work (320×240)
  ▼
  cv::resize(dst, result, Size(460, 360))
  │ 320×240 → 460×360
  │ 耗时：12-24ms
  ▼
m_displayQueue->push(std::move(result))  // 零拷贝转移到队列
  ▼
数据现在在 displayQueue 中
格式：cv::Mat BGR 460×360 CV_8UC3（497KB）
```

### 4.4 阶段 4：主线程显示

```
文件：mainwindow.cpp — displayTimer 回调（每 16ms）

m_displayQueue.latest(mat)               // 取最新帧，跳过中间帧
  │ mat = 浅拷贝（共享数据）
  ▼
cv::cvtColor(mat, rgb, COLOR_BGR2RGB)    // 整个管线唯一的颜色转换
  │ BGR → RGB（Qt 显示需要 RGB）
  ▼
QImage frame(rgb.data, cols, rows, step, Format_RGB888)
  │ 包装为 QImage，不拷贝
  ▼
frame = frame.copy()                     // 深拷贝，脱离 rgb 的内存
  ▼
（如果旋转）frame.transformed + frame.scaled
  ▼
（如果录像）cv::resize(mat, ...) → m_videoWriter.write
  │ 直接用 BGR Mat 写入，不需要 QImage→Mat 转换
  ▼
m_displayImage = frame                   // 保存供 paintEvent 使用
update()                                 // 请求重绘
  ▼
paintEvent(QPaintEvent*)                 // Qt 自动调用
  │ painter.fillRect(黑色背景)
  │ painter.drawImage(居中绘制)
  │ painter.drawText(时间戳 OSD)
  │ painter.drawEllipse(录像红点)
  ▼
屏幕显示
```

### 4.5 数据格式变化总结

```
OV5640 Sensor
  │ RAW Bayer → 硬件 ISP → YUYV 4:2:2
  ▼
V4L2 mmap buffer        YUYV  640×480×2  = 614KB
  │ cv::cvtColor
  ▼
captureQueue             BGR   640×480×3  = 921KB   ← std::move，零拷贝
  │ pop
  ▼
ImageProcessor::process  BGR   320×240×3  = 230KB   ← 内部降分辨率
  │ cv::resize
  ▼
displayQueue             BGR   460×360×3  = 497KB   ← std::move，零拷贝
  │ cv::cvtColor + QImage copy
  ▼
paintEvent               RGB   460×360    = 497KB   ← QImage Format_RGB888
  ▼
屏幕 (linuxfb)           800×480 framebuffer
```

## 五、FrameQueue 详解

### 5.1 为什么需要队列

三个线程速度不同，需要队列做缓冲：

```
采集线程：~33ms/帧（~30fps 产出）
处理线程：~45ms/帧（不开算法）或 ~80ms/帧（开 CLAHE）
主线程：  16ms 轮询一次

采集比处理快 → captureQueue 缓冲，处理线程按自己的速度取
处理比显示慢 → displayQueue 缓冲，主线程取最新帧跳过中间帧
```

### 5.2 核心接口

```cpp
template<int N>  // N = 槽位数量，当前 N=4
class FrameQueue {
    cv::Mat m_frames[N];           // 固定大小数组，不动态分配
    std::atomic<int> m_writeIdx;   // 生产者写位置
    std::atomic<int> m_readIdx;    // 消费者读位置

    // 拷贝版本 push（兼容，当前未使用）
    void push(const cv::Mat &frame);
    //   m_frames[w] = frame.clone()   ← 深拷贝 900KB

    // 移动版本 push（P16 零拷贝，当前使用）
    void push(cv::Mat &&frame);
    //   m_frames[w] = std::move(frame)  ← 转移指针，~0ms

    // 消费者取一帧（处理线程用）
    bool pop(cv::Mat &frame);
    //   frame = m_frames[r]  ← 浅拷贝，引用计数 +1

    // 消费者取最新帧，跳过中间帧（主线程用）
    bool latest(cv::Mat &frame);
    //   frame = m_frames[newest]  ← 跳到最新位置
};
```

### 5.3 队列满时的行为

```
队列容量 4，写入第 5 帧时：

写入前：[帧1] [帧2] [帧3] [帧4]   ← 满了
         ↑r                  ↑w

push 时：移动 readIdx 丢弃帧1
        [帧1] [帧2] [帧3] [帧4]
               ↑r              ↑w

写入帧5：[帧5] [帧2] [帧3] [帧4]
               ↑r    ↑w

帧1 被覆盖（丢弃最旧帧），保证生产者不阻塞。
对于实时视频，丢旧帧是正确的——显示最新画面比显示完整序列更重要。
```

## 六、关键函数速查表

### 6.1 采集线程 (v4l2capture.cpp)

| 函数 | 参数 | 功能 | 耗时 |
|---|---|---|---|
| `open()` | 无 | 初始化 V4L2：查询能力→设置格式→申请 buffer→mmap | 启动时一次 |
| `run()` | 无 | 采集主循环：grabFrame→cvtColor→push | 持续运行 |
| `grabFrame(data, size, bufIdx)` | 输出指针 | select 等待 + DQBUF 取 buffer | ~0ms（等待不算） |
| `returnBuffer(bufIdx)` | buffer 索引 | QBUF 归还 buffer | ~0ms |
| `setAutoWhiteBalance(bool)` | 开/关 | ioctl 控制 OV5640 AWB | ~0ms |
| `setAutoExposure(bool)` | 开/关 | ioctl 控制 OV5640 AE | ~0ms |
| `setContrast(int)` | 0-255 | ioctl 控制 OV5640 对比度 | ~0ms |
| `setSaturation(int)` | 0-255 | ioctl 控制 OV5640 饱和度 | ~0ms |
| `pause()` | 无 | m_running=false，线程继续但不处理帧 | ~0ms |
| `resume()` | 无 | m_running=true，恢复处理 | ~0ms |
| `stop()` | 无 | m_quit=true，线程退出 | ~0ms |

### 6.2 处理线程 (processthread.h)

| 函数 | 参数 | 功能 | 耗时 |
|---|---|---|---|
| `run()` | 无 | 处理主循环：pop→process/resize→push | 持续运行 |
| `pause()` / `resume()` / `stop()` | 无 | 同采集线程 | ~0ms |
| config 成员 | ImageProcessor::Config | 主线程每 16ms 同步 UI 参数到此 | — |

### 6.3 图像处理 (imageprocessor.cpp)

| 函数 | 输入 | 输出 | 功能 | 耗时(320×240) |
|---|---|---|---|---|
| `process(src, dst, cfg, map1, map2)` | BGR 640×480 | BGR 320×240 | 管线入口，按顺序执行所有启用的算法 | 取决于开启的算法 |
| `applyCLAHE(src, dst, clip, tile)` | BGR | BGR | Lab 空间 L 通道自适应均衡化 | ~28ms |
| `applyDehaze(src, dst, omega, r)` | BGR | BGR | 暗通道先验去雾 | ~87ms |
| `applyWhiteBalance(src, dst)` | BGR | BGR | 灰度世界法白平衡（当前不调用） | ~9ms |
| `applySharpen(src, dst, sigma, amt)` | BGR | BGR | USM 锐化 | ~7ms |
| `applyDenoise(src, dst, d, sc, ss)` | BGR | BGR | 双边滤波降噪（内部缩小处理） | ~55ms |
| `applyEdgeDetect(src, dst)` | BGR | BGR | Canny 边缘检测 | ~19ms |
| `applyThreshold(src, dst, thresh)` | BGR | BGR | 二值化阈值分割 | ~1ms |
| `calcSharpness(src)` | BGR/Gray | double | 拉普拉斯方差清晰度评估 | ~2ms |

### 6.4 主线程 (mainwindow.cpp)

| 函数/回调 | 触发方式 | 功能 |
|---|---|---|
| `setupUi()` | 构造函数 | 创建所有 UI 控件（按钮、滑块、面板） |
| `setupVideoSource()` | 构造函数 | 初始化视频源、创建处理线程、启动 displayTimer |
| displayTimer 回调 | 每 16ms | 同步 UI 参数→取最新帧→BGR→RGB→QImage→update |
| `paintEvent(event)` | update() 后 | 绘制视频帧 + OSD（时间戳、录像标识） |
| `onCapturePhoto()` | 拍照按钮 | cv::imwrite 保存 BGR Mat 到 captures/ |
| `onToggleRecord()` | 录像按钮 | 开始/停止 VideoWriter |
| `onToggleFreeze()` | 冻结按钮 | 从 8 帧缓存中选最清晰帧 |
| `onRotateImage()` | 旋转按钮 | m_rotateAngle += 90 |
| `switchToMode(mode)` | 模式按钮 | 0=调试 1=临床 2=编辑，控制面板切换和采集暂停/恢复 |
| `onSelectImage()` | 编辑模式 | 弹出缩略图选择弹窗 |
| `onEditParamChanged()` | 编辑模式滑块 | 对静态图实时预览处理效果 |
| `updateFps()` | 每 1s | 计算并显示帧率 |
