# 系统代码调用链详解 — 第四部分：主线程显示与交互

## 一、主线程的职责

主线程是 Qt 事件循环所在的线程，负责：
1. 从 displayQueue 取最新帧并显示
2. 同步 UI 参数到处理线程
3. 响应用户交互（按钮、滑块、模式切换）
4. 拍照、录像、冻结、旋转
5. 图像编辑模式的离线处理

```
displayTimer (16ms) → 取帧 → BGR→RGB → QImage → paintEvent → 屏幕
按钮点击 → onCapturePhoto / onToggleRecord / ...
```

## 二、显示流程（displayTimer 回调）

### 2.1 完整调用链

每 16ms 触发一次（≈60Hz 轮询，但实际显示帧率取决于 displayQueue 中有没有新帧）：

```cpp
connect(m_displayTimer, &QTimer::timeout, [this]() {
    // ① 同步 UI 参数到处理线程
    if (!m_clinicalMode) {
        m_processThread->config.whiteBalance = false;
        m_processThread->config.clahe = m_chkClahe->isChecked();
        m_processThread->config.claheClipLimit = m_sliderClaheClip->value() / 10.0;
        m_processThread->config.undistort = m_chkUndistort->isChecked() && m_undistortReady;
        m_processThread->config.dehaze = m_chkDehaze->isChecked();
        m_processThread->config.sharpen = false;     // 通用算法固定关闭
        m_processThread->config.denoise = false;
        m_processThread->config.edgeDetect = false;
        m_processThread->config.threshold = false;
    } else {
        m_processThread->config = m_procConfig;      // 临床模式用预设参数
    }

    // ② 从 displayQueue 取最新帧
    cv::Mat mat;
    if (m_displayQueue.latest(mat)) {
        m_latestRawMat = mat;    // 保存 BGR Mat 用于拍照/录像
        m_frameCount++;

        // ③ BGR → RGB → QImage（整个管线唯一的颜色转换）
        cv::Mat rgb;
        cv::cvtColor(mat, rgb, cv::COLOR_BGR2RGB);
        QImage frame(rgb.data, rgb.cols, rgb.rows, rgb.step,
                     QImage::Format_RGB888);
        frame = frame.copy();    // 深拷贝，脱离 rgb 内存

        // ④ 旋转（如果需要）
        if (m_rotateAngle != 0) {
            QTransform transform;
            transform.rotate(m_rotateAngle);
            frame = frame.transformed(transform, Qt::FastTransformation);
            frame = frame.scaled(m_videoRect.size(), Qt::KeepAspectRatio,
                                 Qt::FastTransformation);
        }

        // ⑤ 录像（直接用 BGR Mat，不需要 QImage→Mat 转换）
        if (m_recording && m_videoWriter.isOpened()) {
            cv::Mat resized;
            cv::resize(mat, resized,
                       cv::Size(m_videoRect.width(), m_videoRect.height()));
            m_videoWriter.write(resized);
        }

        // ⑥ 触发重绘
        m_displayImage = frame;
        update();                // 请求 Qt 调用 paintEvent
    }
});
```

### 2.2 数据变换过程

```
displayQueue.latest(mat)     cv::Mat BGR 460×360
  │
  ▼ cv::cvtColor(BGR→RGB)   cv::Mat RGB 460×360
  │
  ▼ QImage(rgb.data, ...)   QImage RGB888 460×360（包装，不拷贝）
  │
  ▼ frame.copy()            QImage RGB888 460×360（深拷贝，脱离 rgb）
  │
  ▼ （旋转/缩放）            QImage RGB888（尺寸可能变化）
  │
  ▼ m_displayImage = frame
  │
  ▼ update() → paintEvent
  │   painter.drawImage()    绘制到 framebuffer → 屏幕
```

### 2.3 为什么需要 frame.copy()

```cpp
cv::Mat rgb;
cv::cvtColor(mat, rgb, cv::COLOR_BGR2RGB);
QImage frame(rgb.data, ...);   // frame 的像素数据指向 rgb 的内存
frame = frame.copy();          // 深拷贝，frame 拥有自己的内存
```

如果不 copy，`rgb` 是局部变量，lambda 结束后被销毁，`frame` 的数据指针变成悬空指针。`frame.copy()` 让 QImage 拥有独立的内存，不依赖 `rgb` 的生命周期。

## 三、paintEvent（窗口绘制）

```cpp
void MainWindow::paintEvent(QPaintEvent *event) {
    QMainWindow::paintEvent(event);    // 父类绘制（背景等）
    QPainter painter(this);

    // 视频区域黑色背景
    painter.fillRect(m_videoRect, Qt::black);

    if (m_displayImage.isNull()) return;

    // 居中绘制视频帧
    int x = m_videoRect.x() + (m_videoRect.width() - m_displayImage.width()) / 2;
    int y = m_videoRect.y() + (m_videoRect.height() - m_displayImage.height()) / 2;
    painter.drawImage(x, y, m_displayImage);

    // OSD：左上角时间戳
    painter.setPen(Qt::yellow);
    painter.drawText(x + 8, y + 16,
                     QDateTime::currentDateTime().toString("yyyy-MM-dd hh:mm:ss"));

    // OSD：录像标识
    if (m_recording) {
        painter.setPen(Qt::red);
        painter.setBrush(Qt::red);
        painter.drawEllipse(x + 8, y + 24, 8, 8);   // 红色圆点
        painter.drawText(x + 20, y + 32, "REC");
    }
}
```

Qt 的绘制机制：`update()` 不会立即绘制，而是在事件队列中放一个重绘请求。Qt 在合适时机（当前事件处理完后）自动调用 `paintEvent`。多次 `update()` 会被合并为一次 `paintEvent`。

## 四、用户交互

### 4.1 拍照（onCapturePhoto）

```
按钮点击 → onCapturePhoto()
  │
  ▼ 检查 m_latestRawMat 是否为空
  │
  ▼ 生成文件名：{savePath}/captures/20250714_153022.jpg
  │
  ▼ cv::imwrite(filename, m_latestRawMat)
  │   直接用 BGR Mat 保存（OpenCV imwrite 接受 BGR）
  │   保存的是处理线程输出的 460×360 图像
  │
  ▼ 状态栏显示 "拍照: ..."
```

### 4.2 录像（onToggleRecord）

```
开始录像：
  ▼ VideoWriter::open(filename, MJPG, 25fps, Size(460, 360))
  ▼ m_recording = true
  ▼ 之后每帧 displayTimer 中：
      cv::resize(mat, resized, Size(460, 360))
      m_videoWriter.write(resized)    ← 直接用 BGR Mat

停止录像：
  ▼ m_recording = false
  ▼ m_videoWriter.release()
```

### 4.3 冻结（onToggleFreeze）

```
冻结：
  ▼ m_frozen = true
  ▼ 遍历 m_frameBuffer（最近 8 帧）
  │   对每帧计算清晰度：ImageProcessor::calcSharpness(mat)
  │   选得分最高的帧
  ▼ 显示最清晰帧
  ▼ 状态栏显示 "已冻结 (最佳帧 3/8, 清晰度 156.7)"

解冻：
  ▼ m_frozen = false
  ▼ 清空 m_frameBuffer
```

### 4.4 旋转（onRotateImage）

```
▼ m_rotateAngle = (m_rotateAngle + 90) % 360
  0° → 90° → 180° → 270° → 0°

旋转在 displayTimer 回调中执行：
  QTransform transform;
  transform.rotate(m_rotateAngle);
  frame = frame.transformed(transform);
  frame = frame.scaled(m_videoRect.size());  // 旋转后尺寸变化，重新缩放
```

## 五、三模式切换（switchToMode）

### 5.1 模式定义

| mode | 名称 | 面板 | 采集状态 | 用途 |
|---|---|---|---|---|
| 0 | 调试模式 | 硬件参数 + 内窥镜算法 + 导出预设 | 运行 | 工程师调参 |
| 1 | 临床模式 | 预设下拉框（胃镜/肠镜/支气管镜） | 运行 | 医生使用 |
| 2 | 图像编辑 | 图像选择 + 锐化/降噪/边缘/阈值 + 保存/删除 | **暂停** | 离线处理 |

### 5.2 切换流程

```cpp
void MainWindow::switchToMode(int mode) {
    if (mode == m_currentMode) return;

    // 离开编辑模式 → 恢复采集
    if (m_currentMode == 2 && mode != 2) {
        m_source->resume();
        m_processThread->resume();
        m_displayTimer->start(16);
    }

    m_currentMode = mode;
    m_ctrlStack->setCurrentIndex(mode);   // 切换面板

    // 更新按钮状态：当前模式置灰
    m_btnModeDebug->setEnabled(mode != 0);
    m_btnModeClinical->setEnabled(mode != 1);
    m_btnModeEdit->setEnabled(mode != 2);

    // 编辑模式下隐藏拍照等按钮
    m_btnCapture->setVisible(mode != 2);
    m_btnRecord->setVisible(mode != 2);
    m_btnFreeze->setVisible(mode != 2);
    m_btnRotate->setVisible(mode != 2);

    switch (mode) {
    case 0:  // 调试
        m_clinicalMode = false;
        break;
    case 1:  // 临床
        m_clinicalMode = true;
        onPresetSelected(m_cboPreset->currentIndex());
        break;
    case 2:  // 编辑
        m_clinicalMode = false;
        m_source->pause();              // 暂停采集（线程不退出）
        m_processThread->pause();       // 暂停处理
        m_displayTimer->stop();         // 停止轮询
        break;
    }
}
```

### 5.3 编辑模式的数据流

编辑模式下采集和处理线程都暂停，数据流完全不同：

```
用户点击"图像选择"
  → onSelectImage()
    → 弹出缩略图弹窗（扫描 captures/ 目录）
      → 用户点击一张图
        → m_editSourceImage = QImage(selectedFile)   加载全分辨率原图
        → 显示到视频区域

用户调节滑块
  → onEditParamChanged()
    → cv::Mat src = ImageConvert::qimageToMat(m_editSourceImage)   QImage→Mat
    → ImageProcessor::process(src, dst, cfg)                        算法处理
    → cv::resize(dst, display, 显示尺寸)
    → m_displayImage = ImageConvert::matToQImage(display)           Mat→QImage
    → update()                                                      显示

用户点击"保存"
  → onSaveEditedImage()
    → 对原图应用处理 → 保存为 xxx_edited.jpg
```

注意编辑模式中使用了 `ImageConvert::qimageToMat` 和 `matToQImage`，因为编辑源图是 QImage 格式（从文件加载）。这和实时流的全程 BGR 架构不同——编辑模式是离线处理，不需要优化格式转换。

## 七、FPS 统计

```cpp
// 每 1 秒触发一次
void MainWindow::updateFps() {
    double elapsed = m_fpsTimer.elapsed() / 1000.0;  // 毫秒 → 秒
    if (elapsed > 0) {
        double fps = m_frameCount / elapsed;
        m_lblFps->setText(QString("FPS: %1").arg(fps, 0, 'f', 1));
    }
    m_frameCount = 0;
    m_fpsTimer.restart();
}
```

`m_frameCount` 在 displayTimer 回调中每取到一帧 +1。所以 FPS 统计的是**主线程实际显示的帧率**，不是采集帧率也不是处理帧率。

处理线程的 FPS 由 processthread.h 内部的日志输出（`[PROC 统计] xxxfps`）。

## 八、存储路径

```cpp
// 构造函数中
if (QDir("/mnt/nfs").exists()) {
    m_savePath = "/mnt/nfs";       // 开发板：NFS 共享目录

    m_savePath = ".";              // 虚拟机：当前目录

QDir().mkpath(m_savePath + "/videos");    // 录像目录
```
