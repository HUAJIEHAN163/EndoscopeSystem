# 文件传输与远程测试操作手册

> 宿主机（虚拟机）与开发板之间传输文件、远程执行程序的完整操作指南。

---

## 当前环境信息

| 项目 | 值 |
|------|-----|
| 宿主机 IP | 192.168.0.128 |
| 开发板 IP | 192.168.0.43（WiFi） |
| 开发板 SSH 用户 | debian |
| 开发板 SSH 密码 | temppwd |
| 开发板串口用户 | root（无密码） |
| 连接方式 | WiFi（延迟波动较大，14ms~1132ms） |

> **注意**：开发板的 root 用户无法通过 SSH 登录（密码认证被拒绝），
> 需要用 `debian` 用户登录。如需 root 权限，登录后执行 `su root`。

---

## 方式一：网络传输（scp）

### 前提条件

- 虚拟机网络设为**桥接模式**（VMware → 虚拟机设置 → 网络适配器 → 桥接模式）
- 开发板已连接 WiFi 或有线网络
- 两者在同一网段（192.168.0.x）

### 1. 测试网络连通性

```bash
# 在宿主机上
ping -c 3 192.168.0.43
```

如果 ping 不通，排查：
- 虚拟机是否为桥接模式？（NAT 模式下虚拟机和开发板不在同一网段）
- 开发板 WiFi 是否已连接？（串口终端执行 `ifconfig wlan0` 查看）
- 防火墙是否阻止？（`sudo ufw status`）

### 2. 从宿主机传文件到开发板

```bash
# 传单个文件
sshpass -p temppwd scp /path/to/local/file debian@192.168.0.43:/tmp/

# 传多个文件
sshpass -p temppwd scp file1 file2 file3 debian@192.168.0.43:/tmp/

# 传整个目录（-r 递归）
sshpass -p temppwd scp -r /path/to/local/dir debian@192.168.0.43:/tmp/

# 传大文件时显示进度（-v 详细模式）
sshpass -p temppwd scp -v bigfile debian@192.168.0.43:/tmp/
```

如果没有 sshpass，需要手动输入密码：
```bash
scp /path/to/file debian@192.168.0.43:/tmp/
# 提示输入密码时输入：temppwd
```

### 3. 从开发板拉文件到宿主机

```bash
# 拉单个文件
sshpass -p temppwd scp debian@192.168.0.43:/tmp/remote_file ./

# 拉整个目录
sshpass -p temppwd scp -r debian@192.168.0.43:/home/debian/data ./
```

### 4. 常用传输场景

```bash
# 场景 A：传编译好的程序到开发板
sshpass -p temppwd scp my_app debian@192.168.0.43:/tmp/

# 场景 B：传 OpenCV .so 库到开发板的 /usr/lib（需要 root 权限）
# 先传到 /tmp，再在开发板上用 root 移动
sshpass -p temppwd scp /opt/opencv_arm/lib/libopencv_*.so.3.4.16 debian@192.168.0.43:/tmp/
# 然后 SSH 登录开发板，su root 后移动文件（见下方"远程执行"章节）

# 场景 C：从开发板拉回日志或截图
sshpass -p temppwd scp debian@192.168.0.43:/tmp/screenshot.png ./
```

### 5. 首次连接的提示

第一次 scp/ssh 连接时会提示确认主机指纹：
```
The authenticity of host '192.168.0.43' can't be established.
Are you sure you want to continue connecting (yes/no)?
```

输入 `yes` 即可。或者加 `-o StrictHostKeyChecking=no` 跳过：
```bash
sshpass -p temppwd scp -o StrictHostKeyChecking=no file debian@192.168.0.43:/tmp/
```

## 方式二：USB 存储传输（U盘 / SD 卡读卡器）

### 适用场景

- WiFi 不稳定或未配置网络时
- 需要传输大文件（几百 MB 以上）时，比 WiFi scp 快得多

### 1. 在宿主机上准备文件

```bash
# 把要传的文件拷贝到 U 盘
# 先把 U 盘插到电脑（物理机），然后在 VMware 中连接到虚拟机
# VMware 菜单 → 虚拟机 → 可移动设备 → 找到你的 U 盘 → 连接

# 查看 U 盘设备
lsblk
# 通常显示为 /dev/sdb1 或 /dev/sdc1

# 挂载 U 盘（如果没有自动挂载）
sudo mkdir -p /mnt/usb
sudo mount /dev/sdb1 /mnt/usb

# 拷贝文件到 U 盘
cp my_app /mnt/usb/
cp /opt/opencv_arm/lib/libopencv_*.so* /mnt/usb/

# 安全弹出
sync
sudo umount /mnt/usb
```

### 2. 在开发板上读取 U 盘

```bash
# 把 U 盘从电脑拔出，插到开发板的 USB 口

# 在开发板串口终端中操作（root 用户）
# 查看是否识别到 U 盘
lsblk
# 或
dmesg | tail -20
# 应看到类似：sd 0:0:0:0: [sda] ... 的日志

# 挂载 U 盘
mkdir -p /mnt/usb
mount /dev/sda1 /mnt/usb
# 注意：设备名可能是 sda1、sdb1，以 dmesg 输出为准

# 拷贝文件
cp /mnt/usb/my_app /tmp/
cp /mnt/usb/libopencv_*.so* /usr/lib/

# 弹出 U 盘
sync
umount /mnt/usb
```

### 3. 注意事项

- U 盘文件系统建议用 **FAT32** 或 **ext4**，开发板都能识别
- NTFS 格式可能需要安装 `ntfs-3g`，开发板上不一定有
- 拷贝完记得 `sync` 再拔，防止数据丢失

## 方式三：NFS 共享目录（开发效率最高）

### 适用场景

- 频繁编译、频繁传输时，省去每次 scp 的操作
- 宿主机编译完，开发板上直接就能运行

### 1. 宿主机配置 NFS 服务（已安装 nfs-kernel-server）

```bash
# 创建共享目录
mkdir -p ~/nfs_share

# 配置导出（只需执行一次）
echo "/home/han/nfs_share *(rw,sync,no_root_squash)" | sudo tee -a /etc/exports
sudo exportfs -arv
sudo systemctl restart nfs-kernel-server

# 验证导出
showmount -e localhost
# 应显示：/home/han/nfs_share *
```

### 2. 开发板挂载 NFS

```bash
# 在开发板上（root 用户）
mkdir -p /mnt/nfs
mount -t nfs 192.168.0.128:/home/han/nfs_share /mnt/nfs -o nolock

# 验证
ls /mnt/nfs
# 应能看到宿主机 ~/nfs_share 中的文件
```

### 3. 使用方式

```bash
# 宿主机上：把编译产物放到共享目录
cp my_app ~/nfs_share/

# 开发板上：直接运行
/mnt/nfs/my_app
# 不需要 scp，实时同步！
```

### 4. 开机自动挂载（可选）

在开发板的 `/etc/fstab` 中添加：
```
192.168.0.128:/home/han/nfs_share /mnt/nfs nfs nolock,defaults 0 0
```

## 远程执行程序

### 1. SSH 登录开发板

```bash
# 交互式登录
sshpass -p temppwd ssh debian@192.168.0.43

# 登录后如需 root 权限
su root
# 密码：root（串口终端的 root 密码）
```

### 2. 远程执行单条命令（不登录）

```bash
# 执行一条命令并返回结果
sshpass -p temppwd ssh debian@192.168.0.43 "命令"

# 示例：查看开发板系统信息
sshpass -p temppwd ssh debian@192.168.0.43 "uname -a"

# 示例：运行测试程序
sshpass -p temppwd ssh debian@192.168.0.43 "/tmp/test_level1"

# 示例：执行多条命令（用 && 连接）
sshpass -p temppwd ssh debian@192.168.0.43 "chmod +x /tmp/my_app && /tmp/my_app"
```

### 3. 远程执行需要设置环境变量的程序

```bash
# 如果程序依赖额外的 .so 库（如 OpenCV），需要设置 LD_LIBRARY_PATH
sshpass -p temppwd ssh debian@192.168.0.43 \
    "export LD_LIBRARY_PATH=/tmp:\$LD_LIBRARY_PATH && /tmp/test_level3"

# 如果是 Qt 程序，需要设置显示后端
sshpass -p temppwd ssh debian@192.168.0.43 \
    "export QT_QPA_PLATFORM=linuxfb:fb=/dev/fb0 && /tmp/qt_app"
```

### 4. 完整的"编译→传输→运行"流程示例

```bash
# Step 1: 在宿主机上交叉编译
export TC=/opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf/bin
$TC/arm-linux-gnueabihf-g++ my_app.cpp -o my_app \
    -I/opt/opencv_arm/include \
    -L/opt/opencv_arm/lib \
    -lopencv_core -lopencv_imgproc

# Step 2: 传到开发板
sshpass -p temppwd scp my_app debian@192.168.0.43:/tmp/

# Step 3: 远程运行
sshpass -p temppwd ssh debian@192.168.0.43 \
    "chmod +x /tmp/my_app && /tmp/my_app"
```

### 5. 部署 .so 库到开发板（一次性操作）

开发板上运行程序前，需要先把依赖的 .so 库部署到开发板：

```bash
# 传 OpenCV 库到开发板
sshpass -p temppwd scp /opt/opencv_arm/lib/libopencv_*.so.3.4.16 debian@192.168.0.43:/tmp/

# SSH 登录开发板，移动到系统库目录
sshpass -p temppwd ssh debian@192.168.0.43 "su -c '\
    cp /tmp/libopencv_*.so.3.4.16 /usr/lib/ && \
    cd /usr/lib && \
    for f in libopencv_*.so.3.4.16; do \
        base=\${f%.3.4.16}; \
        ln -sf \$f \$base; \
        ln -sf \$f \${base%.so.3.4}.so; \
    done && \
    ldconfig && \
    echo done' root"

# 部署完成后，程序运行时就不需要设置 LD_LIBRARY_PATH 了
```

## 常见问题

### Q: scp 传输很慢？
WiFi 延迟波动大（14ms~1132ms），大文件建议：
- 用有线网络连接开发板（eth0）
- 用 U 盘传输
- 用 NFS 共享目录

### Q: SSH 连接超时？
```bash
# 加超时参数
sshpass -p temppwd ssh -o ConnectTimeout=15 debian@192.168.0.43 "命令"
```

### Q: 提示 "Host key verification failed"？
```bash
# 清除旧的主机密钥（开发板重刷系统后会变）
ssh-keygen -R 192.168.0.43
```

### Q: 想免密码登录？
```bash
# 生成密钥（如果还没有）
ssh-keygen -t ed25519 -N ""

# 拷贝公钥到开发板
sshpass -p temppwd ssh-copy-id debian@192.168.0.43

# 之后就可以直接 ssh/scp，不需要 sshpass 了
ssh debian@192.168.0.43
```

### Q: 程序运行报 "No such file or directory"（文件明明存在）？
通常是动态链接器找不到。检查：
```bash
# 在开发板上
file /tmp/my_app          # 确认是 ARM 格式
ldd /tmp/my_app           # 查看缺少哪些 .so
```

### Q: 程序运行报 "GLIBC_x.xx not found"？
说明编译时用的工具链 glibc 版本高于开发板。确保使用 Linaro gcc 7.5 工具链编译：
```bash
# 正确 ✅
/opt/toolchain/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf/bin/arm-linux-gnueabihf-g++ ...

# 错误 ❌（Ubuntu 24.04 自带的，glibc 太新）
arm-linux-gnueabihf-g++ ...
```
