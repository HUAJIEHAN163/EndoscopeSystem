# 线程与类的关系

## 一、核心概念

**线程是执行代码的"工人"，类和对象是"工具"。**

- 线程不拥有对象，对象不属于线程
- 对象属于创建它的那块内存（通常是某个类的成员变量）
- 任何线程都可以访问任何对象，只要有指针或引用
- 代码在哪个线程中被调用，就在那个线程的 CPU 时间片里执行

## 二、线程的生命周期

### 2.1 创建和启动

```cpp
// QThread 方式（我们项目）
m_processThread = new ProcessThread(...);  // 创建对象（主线程中）
m_processThread->start();                  // 启动线程 → 内部创建新线程 → 调用 run()

// std::thread 方式（无 Qt）
std::thread t(&MyClass::run, this);        // 创建并立即启动
```

`start()` 调用后**立即返回**，不会等 run() 执行完。之后两个线程同时运行。

### 2.2 暂停和恢复

```cpp
// 我们项目的做法：用 atomic 标志控制
void pause()  { m_running = false; }   // 主线程调用
void resume() { m_running = true; }    // 主线程调用

// run() 中检查标志
void run() override {
    while (!m_quit) {
        if (!m_running) {          // 暂停时
            QThread::msleep(10);   // 睡眠，不占 CPU
            continue;              // 跳过处理
        }
        // ... 正常处理 ...
    }
}
```

暂停不是真的"冻结"线程，而是让线程在循环中空转等待。线程仍然活着，只是不做有用的工作。

### 2.3 停止和退出

```cpp
// 停止：设置退出标志
void stop() { m_quit = true; m_running = false; }

// run() 中检查退出标志
void run() override {
    while (!m_quit) {    // m_quit 为 true 时退出循环
        // ...
    }
    // run() 返回后线程自动结束
}

// 等待线程真正结束
m_processThread->stop();   // 设置标志
m_processThread->wait();   // 阻塞等待 run() 返回（QThread）
// 或
t.join();                  // 阻塞等待（std::thread）
```

### 2.4 为什么用两个标志（m_running 和 m_quit）

```
m_running = false, m_quit = false  → 暂停（线程活着，不处理）
m_running = true,  m_quit = false  → 正常运行
m_running = false, m_quit = true   → 退出（线程结束）
```

V4L2 采集线程暂停时仍然需要 grabFrame + returnBuffer 保持视频流活跃（否则 STREAMOFF 后无法恢复），所以不能用 stop/start，只能用 pause/resume。

## 三、类的三种使用模式

### 3.1 有线程的类（继承 QThread）

自己拥有一个线程，start() 后在独立线程中执行 run()。

```cpp
class ProcessThread : public QThread {
    void run() override {
        // 在独立线程中执行
        while (!m_quit) { ... }
    }
};

// 使用
m_processThread->start();   // 启动线程
m_processThread->stop();    // 停止线程
```

我们项目中有两个：V4l2Capture（采集线程）、ProcessThread（处理线程）。

### 3.2 普通类（需要创建对象）

没有线程，谁调用它的方法，就在谁的线程中执行。

```cpp
class ImageConvert {
public:
    static QImage matToQImage(const cv::Mat &mat);   // 静态方法
    static cv::Mat qimageToMat(const QImage &image);
};

// 在处理线程中调用 → 在处理线程中执行
ImageConvert::matToQImage(mat);

// 在主线程中调用 → 在主线程中执行
ImageConvert::qimageToMat(image);
```

### 3.3 全 static 类（不需要创建对象）

所有方法都是 static，通过类名直接调用，不需要 new。

```cpp
class ImageProcessor {
public:
    static void process(...);      // 全部是 static
    static void applyCLAHE(...);
    static void applyDehaze(...);
};

// 直接调用，不需要对象
ImageProcessor::process(src, dst, cfg);
```

**为什么 ImageProcessor 设计成全 static**：

它没有状态——所有算法都是"输入一张图 + 参数，输出一张图"，不需要在帧之间记住任何东西。不需要对象就不创建对象，代码更简单。

**对比**：

| | 全 static 类 | 普通类 |
|---|---|---|
| 创建对象 | 不需要 | 需要 new 或声明 |
| 调用方式 | `类名::方法()` | `对象.方法()` 或 `指针->方法()` |
| 内部状态 | 没有（每次调用独立） | 有（成员变量在调用之间保持） |
| 适合场景 | 纯工具函数 | 需要维护状态的对象 |
| 例子 | ImageProcessor、ImageConvert | ProcessThread、V4l2Capture |

## 四、对象归属与跨线程访问

### 4.1 对象属于谁

对象属于创建它的那块内存，不属于任何线程：

```cpp
class MainWindow {
    FrameQueue<4> m_captureQueue;           // 实体，属于 MainWindow
    ProcessThread *m_processThread;          // 指针，指向 new 出来的对象
};
```

### 4.2 多个线程访问同一个对象

通过指针，多个线程可以访问同一个对象：

```
MainWindow 拥有 m_captureQueue（实体）
  │
  ├── 采集线程通过指针访问：m_source->m_captureQueue->push(...)
  ├── 处理线程通过指针访问：m_captureQueue->pop(...)
  └── 主线程直接访问：m_captureQueue.size()

三个线程操作的是同一个队列，不是三个副本
```

### 4.3 跨线程访问的安全规则

**安全的**（我们项目中的用法）：

| 模式 | 例子 | 为什么安全 |
|---|---|---|
| 一个线程写简单类型，另一个读 | 主线程写 config.clahe，处理线程读 | bool/int 单字段读写在 ARM 上是原子的 |
| 用 atomic 变量 | m_running、m_quit | atomic 保证线程间可见性 |
| 用无锁队列 | FrameQueue 的 push/pop | 单生产者单消费者，用 atomic 索引保证安全 |
| 只读共享 | undistortMap1/Map2 | 初始化后不再修改，多线程读没问题 |

**危险的**（需要加锁或避免）：

| 模式 | 例子 | 为什么危险 |
|---|---|---|
| 两个线程同时写同一个变量 | 两个线程都写 config.clahe | 结果不确定 |
| 读写复杂对象 | 一个线程写 std::vector，另一个读 | vector 内部有多个指针，可能读到半写状态 |
| 一个线程释放，另一个还在用 | 主线程 delete 对象，其他线程还在访问 | 崩溃 |

### 4.4 我们项目中的跨线程数据流

```
主线程写 → 处理线程读（通过 config 成员）：
  主线程 displayTimer: m_processThread->config.clahe = true;
  处理线程 run():      if (config.clahe) applyCLAHE(...);
  安全：bool 单字段，单写单读

采集线程写 → 处理线程读（通过 captureQueue）：
  采集线程: m_captureQueue->push(std::move(bgr));
  处理线程: m_captureQueue->pop(frame);
  安全：无锁队列，atomic 索引保证

处理线程写 → 主线程读（通过 displayQueue）：
  处理线程: m_displayQueue->push(std::move(result));
  主线程:   m_displayQueue.latest(mat);
  安全：同上

主线程调用 → 处理线程响应（通过 atomic 标志）：
  主线程:   m_processThread->pause();   // 写 m_running = false
  处理线程: if (!m_running) continue;   // 读 m_running
  安全：atomic<bool>
```

## 五、static 关键字的三种含义

`static` 在 C++ 中有三种不同的含义，容易混淆：

### 5.1 static 成员方法（类级别）

属于类而不是对象，通过类名调用：

```cpp
class ImageProcessor {
    static void process(...);   // 属于 ImageProcessor 类
};

ImageProcessor::process(...);   // 不需要对象
```

### 5.2 static 局部变量（函数级别）

函数内部的变量，只初始化一次，后续调用保持上次的值：

```cpp
void ImageProcessor::applyCLAHE(...) {
    static cv::Ptr<cv::CLAHE> clahe = cv::createCLAHE();  // 只创建一次
    // 后续调用复用同一个 clahe 对象
}
```

### 5.3 static 全局变量/函数（文件级别）

限制作用域在当前文件内，其他文件看不到。我们项目中没有用到这种。

```cpp
// file_a.cpp
static int counter = 0;    // 只在 file_a.cpp 中可见

// file_b.cpp
static int counter = 0;    // 和 file_a.cpp 的 counter 是两个不同的变量
```

## 六、我们项目中所有类的分类

| 类 | 类型 | 有线程吗 | 怎么调用 |
|---|---|---|---|
| V4l2Capture | 继承 QThread | ✅ 采集线程 | start() 启动 |
| FileCapture | 继承 QThread | ✅ 采集线程 | start() 启动 |
| ProcessThread | 继承 QThread | ✅ 处理线程 | start() 启动 |
| MainWindow | 继承 QMainWindow | ❌ 在主线程中 | Qt 事件循环驱动 |
| ImageProcessor | 全 static | ❌ 无线程 | 类名::方法() |
| ImageConvert | 全 static | ❌ 无线程 | 类名::方法() |
| PresetManager | 全 static | ❌ 无线程 | 类名::方法() |
| FrameQueue | 模板类 | ❌ 无线程 | 被多个线程通过指针共享访问 |
| VideoSource | 抽象基类 | ✅（子类实现） | 定义接口，不直接使用 |
