# 03 图像处理算法原理

## 一、USM 锐化（当前实现）

### 原理

锐化 = 原图 + 强度 × 高频细节

高频细节 = 原图 - 模糊图

```
dst = src + amount × (src - blur(src))
    = src × (1 + amount) - blur(src) × amount
```

```cpp
// 项目实现
cv::GaussianBlur(src, blurred, cv::Size(3, 3), 0);
cv::addWeighted(src, 1.0 + amount, blurred, -amount, 0, dst);
```

### 问题

- 噪声也是高频信号，会被一起放大
- 平坦区域和边缘区域用同样的强度

## 二、自适应边缘锐化（P14 目标）

### 原理

只在边缘区域锐化，平坦区域保持不变：

```
1. 计算边缘强度图（Laplacian）
2. 归一化为权重掩码 mask（0=平坦，1=边缘）
3. 混合：dst = src × (1-mask) + usm × mask
```

### 为什么用 Laplacian 做边缘检测

Laplacian 计算二阶导数，对亮度变化敏感：

```
平坦区域：像素值 [100, 100, 100] → Laplacian ≈ 0
边缘区域：像素值 [100, 100, 200] → Laplacian ≈ 大值
噪声：    像素值 [100, 105, 100] → Laplacian ≈ 小值（比边缘小）
```

通过归一化后，边缘处 mask 接近 1（锐化），噪声处 mask 接近 0（不锐化）。

### 优化方向

| 瓶颈 | 优化方法 |
|---|---|
| 浮点混合慢 | 改用整数运算 |
| mask 精度过高 | 降低 mask 分辨率（缩小再放大） |
| 逐像素运算 | NEON 向量化 |

## 三、CLAHE 自适应直方图均衡化

### 普通直方图均衡化的问题

全局均衡化会过度增强某些区域：

```
原图：左半部分暗，右半部分亮
全局均衡化后：左半部分变亮了，但右半部分过曝
```

### CLAHE 的改进

1. 把图像分成小块（如 8×8 个块）
2. 每个块独立做直方图均衡化
3. 用 clipLimit 限制对比度增强的幅度（防止过度增强）
4. 块与块之间做双线性插值，避免块边界出现明显分界线

```
clipLimit 的作用：
  clipLimit 小 → 增强弱，接近原图
  clipLimit 大 → 增强强，对比度高但可能出现噪声放大
  项目默认值 3.0，范围 1.0-8.0
```

### 为什么在 Lab 空间做

```
在 BGR 空间做：三个通道分别均衡化 → 颜色失真
在 Lab 空间做：只均衡化 L（亮度）通道 → 颜色不变，只调亮暗
```

## 四、去雾（暗通道先验）

### 雾的物理模型

```
I(x) = J(x) × t(x) + A × (1 - t(x))

I(x) = 观察到的有雾图像
J(x) = 无雾的清晰图像（我们要恢复的）
t(x) = 透射率（0=全雾，1=无雾）
A    = 大气光（雾的颜色，通常接近白色）
```

### 暗通道先验

观察发现：无雾图像中，大多数局部区域至少有一个颜色通道的值接近 0。

```
暗通道 = 对每个像素，取 RGB 三通道的最小值，再在局部区域取最小值

无雾图像的暗通道 → 接近 0
有雾图像的暗通道 → 远大于 0（因为雾让所有通道都变亮）
```

### 算法步骤

```
1. 计算暗通道：dark = min(min(B, G, R))，再 erode
2. 估计大气光 A：暗通道最亮区域的均值
3. 估计透射率：t = 1 - omega × dark_normalized
4. 恢复清晰图像：J = (I - A) / max(t, 0.1) + A
```

### 项目中的优化

- CV_64F → CV_32F：单精度浮点，速度翻倍
- 去掉多余的 split/merge：直接用 cv::divide 逐通道计算

## 五、白平衡（灰度世界法）

### 原理

假设：图像中所有颜色的平均值应该是灰色（R=G=B）。

如果实际平均值不是灰色，说明有色偏，通过调整各通道增益来补偿：

```
avgB = mean(B通道)
avgG = mean(G通道)
avgR = mean(R通道)
avg  = (avgB + avgG + avgR) / 3

B通道 *= avg / avgB    // 蓝色偏多就压低，偏少就提高
G通道 *= avg / avgG
R通道 *= avg / avgR
```

### 局限性

灰度世界法假设图像整体颜色均匀分布。内窥镜场景中大面积红色黏膜，假设不成立，效果不如硬件 AWB。这就是为什么我们把白平衡卸载给 OV5640 硬件。

## 六、双边滤波降噪

### 普通高斯模糊的问题

高斯模糊对所有像素一视同仁，边缘也会被模糊：

```
原图：  [100, 100, 100, 200, 200, 200]
                         ↑ 边缘
高斯模糊：[100, 100, 133, 167, 200, 200]
                         ↑ 边缘被模糊了
```

### 双边滤波的改进

同时考虑空间距离和颜色差异：
- 空间距离近的像素权重大（和高斯一样）
- 颜色差异小的像素权重大（新增）
- 颜色差异大的像素权重小（边缘两侧不互相影响）

```
原图：  [100, 100, 100, 200, 200, 200]
双边滤波：[100, 100, 100, 200, 200, 200]
                         ↑ 边缘保留了！
```

### 为什么慢

每个像素需要计算邻域内所有像素的颜色差异权重，计算量是高斯模糊的数倍。项目中降到半分辨率处理。

## 七、边缘检测（Canny）

### 步骤

```
1. 高斯模糊（去噪）
2. Sobel 计算 X/Y 方向梯度
3. 计算梯度幅值和方向
4. 非极大值抑制（只保留梯度方向上的局部最大值，细化边缘）
5. 双阈值：
   - 梯度 > 高阈值 → 强边缘（保留）
   - 低阈值 < 梯度 < 高阈值 → 弱边缘（连接到强边缘才保留）
   - 梯度 < 低阈值 → 非边缘（丢弃）
```

## 八、算法管线顺序

项目中 `process()` 的算法执行顺序是有讲究的：

```
白平衡 → CLAHE → 畸变校正 → 去雾 → 锐化 → 降噪 → 边缘检测 → 阈值分割
```

为什么这个顺序：
1. 白平衡先做：后续算法基于正确的颜色
2. CLAHE 在去雾前：先提高对比度，去雾效果更好
3. 锐化在降噪前：先锐化再降噪，避免降噪模糊掉锐化的细节
4. 边缘检测/阈值分割最后：这两个是分析算法，输出是二值图，不能再做其他处理
