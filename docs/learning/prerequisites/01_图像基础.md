# 01 图像基础

## 一、像素与图像表示

### 1.1 什么是像素

图像由像素（pixel）组成，每个像素是一个颜色值：

```
一张 4×3 的图像 = 12 个像素

  列0   列1   列2   列3
行0 [像素] [像素] [像素] [像素]
行1 [像素] [像素] [像素] [像素]
行2 [像素] [像素] [像素] [像素]
```

### 1.2 灰度图

每个像素用 1 个字节（0-255）表示亮度：

```
0 = 纯黑，255 = 纯白，128 = 中灰

[  0] [ 50] [128] [200] [255]
 黑    暗灰   中灰   亮灰   白
```

在 OpenCV 中：`CV_8UC1`（8位无符号，1通道）

### 1.3 彩色图（BGR/RGB）

每个像素用 3 个字节表示，分别是三个颜色通道：

```
BGR 格式（OpenCV 默认）：
像素 = [B=蓝, G=绿, R=红]

纯红色 = [0, 0, 255]      B=0, G=0, R=255
纯绿色 = [0, 255, 0]
纯蓝色 = [255, 0, 0]
白色   = [255, 255, 255]   三通道都满
黑色   = [0, 0, 0]         三通道都空
```

BGR 和 RGB 的区别只是通道顺序：
- OpenCV 用 BGR（历史原因）
- Qt QImage 用 RGB
- 所以需要 cvtColor 转换

在 OpenCV 中：`CV_8UC3`（8位无符号，3通道）

### 1.4 图像在内存中的存储

图像在内存中是一段连续的字节数组，按行存储：

```
灰度图 3×2（3列2行）：
内存：[p00, p01, p02, p10, p11, p12]
       行0          行1

BGR 彩色图 3×2：
内存：[B00,G00,R00, B01,G01,R01, B02,G02,R02, B10,G10,R10, ...]
       像素(0,0)     像素(0,1)     像素(0,2)     像素(1,0)
```

每行的字节数叫 `step`（步长），可能比 `width × channels` 大（内存对齐）。

## 二、色彩空间

### 2.1 RGB / BGR

最直观的颜色表示，三个通道分别控制红绿蓝的强度。

```
优点：直观，显示设备直接使用
缺点：亮度和颜色混在一起，不方便单独处理亮度
```

### 2.2 YUV / YCbCr

将亮度和色度分离：

```
Y  = 亮度（人眼最敏感）
U(Cb) = 蓝色色度
V(Cr) = 红色色度

从 RGB 计算：
Y  =  0.299×R + 0.587×G + 0.114×B
Cb = -0.169×R - 0.331×G + 0.500×B + 128
Cr =  0.500×R - 0.419×G - 0.081×B + 128
```

用途：
- 摄像头输出（YUYV 格式）
- JPEG 压缩（对色度下采样）
- 视频编码（H.264）

### 2.3 Lab

也是亮度和色度分离，但更符合人眼感知：

```
L = 亮度（0-100）
a = 绿色(-128) ↔ 红色(+127)
b = 蓝色(-128) ↔ 黄色(+127)
```

用途：
- CLAHE 在 Lab 空间的 L 通道上做均衡化，只调亮度不影响颜色
- 颜色差异计算（Lab 空间中的欧氏距离更符合人眼感知）

设计过程：
人们发现 RGB 空间中数值差异不等于人眼感知差异：

RGB 空间：
  (200,0,0) 和 (210,0,0) 差 10 → 人眼几乎看不出
  (0,0,200) 和 (0,0,210) 差 10 → 人眼能看出

同样差 10，感知不一样，因为人眼对不同颜色的敏感度不同

于是科学家做了大量人眼实验，测量"人觉得两个颜色差多少"，然后设计了一个非线性变换，使得：

Lab 空间：
  两个颜色的 Lab 数值差 = 人眼感知的颜色差

  差 1 → 刚好能分辨
  差 5 → 明显不同
  差 10 → 差异很大


具体转换：
RGB → Lab：最复杂，需要两步
RGB → XYZ（线性变换）→ Lab（非线性变换，含立方根）

### 2.4 HSV

用色调、饱和度、明度描述颜色，更接近人的直觉：

```
H = 色调（0-179，颜色种类：红→黄→绿→青→蓝→紫）
S = 饱和度（0-255，颜色鲜艳程度）
V = 明度（0-255，亮暗程度）
```

用途：
- 颜色检测（如检测红色病灶区域）
- 比 RGB 更容易做颜色分割

设计过程：
RGB 三个值 → 通过数学公式 → H（什么颜色）S（多鲜艳）V（多亮）

具体转换：
RGB → HSV：三角函数运算
V = max(R, G, B)
S = (V - min(R, G, B)) / V
H = 根据哪个通道最大，用不同的公式算角度


### 2.5 色彩空间转换的开销

每次转换都需要对每个像素做数学运算，640×480 图像有 307,200 个像素：

| 转换 | Cortex-A7 耗时 | 说明 |
|---|---|---|
| BGR → RGB | ~5ms | 只交换通道顺序 |
| BGR → Gray | ~3ms | 加权求和 |
| BGR → Lab | ~10ms | 复杂的非线性变换 |
| BGR → HSV | ~8ms | 三角函数运算 |
| YUYV → BGR | ~15ms | 每2像素解包+矩阵运算 |

这就是为什么 P7.1 要减少颜色转换次数——每次转换都不便宜。

## 三、图像的数学本质

### 3.1 图像是矩阵

一张灰度图就是一个二维矩阵，彩色图是三维矩阵（或三个二维矩阵叠加）：

```
灰度图 = 一个矩阵
         [128, 200, 50 ]
         [100, 150, 80 ]
         [60,  90,  120]

彩色图 = 三个矩阵（通道）
         B通道          G通道          R通道
         [128, 200]     [50,  100]     [80,  150]
         [60,  90 ]     [120, 80 ]     [200, 30 ]
```

### 3.2 卷积（滤波的核心）

卷积是图像处理最基本的操作——用一个小矩阵（核/kernel）在图像上滑动，对覆盖区域做加权求和：

```
3×3 均值滤波核：
[1/9, 1/9, 1/9]
[1/9, 1/9, 1/9]
[1/9, 1/9, 1/9]

对每个像素：新值 = 周围 9 个像素的平均值
效果：图像变模糊（平滑噪声）
```

```
3×3 锐化核（拉普拉斯）：
[ 0, -1,  0]
[-1,  4, -1]
[ 0, -1,  0]

对每个像素：新值 = 中心像素×4 - 上下左右像素之和
效果：突出边缘（相邻像素差异大的地方值大）
```

不同的核实现不同的效果：
- 均值核 → 模糊
- 高斯核 → 高斯模糊（更自然的模糊）
- 拉普拉斯核 → 边缘检测
- Sobel 核 → 方向性边缘检测

### 3.3 直方图

统计图像中每个亮度值出现的次数：

```
一张偏暗的图像：
亮度值：  0   50  100  150  200  255
像素数：  ███ ██  █    ·    ·    ·
          大部分像素集中在暗区

直方图均衡化后：
亮度值：  0   50  100  150  200  255
像素数：  ██  ██  ██   ██   ██   ██
          像素均匀分布，对比度提高
```

CLAHE 就是在局部区域做直方图均衡化，避免全局均衡化导致的过度增强。

### 3.4 梯度

梯度描述像素值变化的快慢，边缘处梯度大，平坦处梯度小：

```
原始像素：  [100, 100, 100, 200, 200, 200]
                              ↑ 边缘
梯度：      [  0,   0, 100, 100,   0,   0]
                         ↑ 梯度大 = 边缘
```

自适应锐化就是利用梯度：梯度大的地方（边缘）锐化，梯度小的地方（平坦）不处理。

## 四、与项目的对应关系

| 概念 | 项目中哪里用到 |
|---|---|
| BGR/RGB | 采集线程 cvtColor，processFrame 中 QImage↔Mat 转换 |
| Lab 色彩空间 | CLAHE 在 L 通道上做均衡化 |
| YUYV | V4L2 摄像头输出格式 |
| 卷积 | GaussianBlur（锐化用）、erode（去雾用） |
| 直方图 | CLAHE 自适应直方图均衡化 |
| 梯度 | Laplacian（自适应锐化的边缘掩码）、Canny（边缘检测） |
| 图像矩阵 | cv::Mat 就是矩阵的封装 |
