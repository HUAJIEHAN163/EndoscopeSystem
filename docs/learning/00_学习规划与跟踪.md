### 通信协议为什么是通用技能

同一个协议在两侧都会用到，理解协议本身是通用的，实现方式不同：

| 协议 | A7 侧（Linux 应用层） | M4 侧（裸机/RTOS） |
|------|----------------------|-------------------|
| UART | /dev/ttySTM0 + termios | HAL_UART_Transmit/Receive |
| I2C | /dev/i2c-x + ioctl | HAL_I2C_Master_Transmit |
| SPI | /dev/spidev + ioctl | HAL_SPI_TransmitReceive |

协议原理学一次，两侧都能用。动手练习主要在 M4 阶段。

### 2.1 C++

**学习方法**：对照项目代码学，遇到不懂的语法当场查，不要先系统学完再看代码。

**学习路径**：
1. 类与对象：看 `MainWindow` 类的定义和使用

3. 虚函数与重写：`paintEvent`、`keyPressEvent` 的 override
4. 构造/析构函数：`MainWindow::MainWindow()`、`~MainWindow()`
5. 指针与 new/delete：Qt 的父子内存管理机制
6. 引用与 const：函数参数中的 `const QImage &image`
7. STL 容器：`std::deque`（帧缓存）、`std::vector`（OpenCV 通道操作）

**目标要求**：
- [ ] 能看懂项目中所有 C++ 语法

**目标要求**：
- [ ] 能实现一个线程安全的环形队列（FrameRingBuffer）
- [ ] 能解释无锁设计的原理和限制（单生产者单消费者）
- [ ] 能把当前两线程架构改为三线程流水线
- [ ] 理解线程优先级对实时性的影响

**参考资源**：《C++ Concurrency in Action》Ch 4-7，或搜索 "lock-free ring buffer C++"

### 2.7 ARM NEON SIMD 加速

**学习方法**：先理解 SIMD 思想（一条指令处理多个数据），再用 YUYV→RGB 转换作为入门练习。

4. 实战：YUYV→RGB 的 NEON 版本，与 OpenCV cvtColor 对比性能
5. 进阶：加速高斯滤波、图像混合等运算

**目标要求**：

**参考资源**：ARM NEON Programmer's Guide，https://developer.arm.com/architectures/instruction-sets/simd-isas/neon

### 2.8 第三方库集成

**学习方法**：理解嵌入式项目中集成第三方库的完整流程，包括算法组提供的 .a/.lib 静态库和 .so 动态库。

**学习路径**：
1. 静态库 vs 动态库的区别：
   - `.a`（Linux）/ `.lib`（Windows）：静态库，编译时嵌入可执行文件
   - `.so`（Linux）/ `.dll`（Windows）：动态库，运行时加载
2. CMake 集成静态库：
   ```cmake
   # 算法组提供了 libalgo.a 和 algo.h
   target_include_directories(endoscope PRIVATE /path/to/algo/include)
   target_link_libraries(endoscope /path/to/algo/lib/libalgo.a)
   ```
3. CMake 集成动态库：
   ```cmake
   target_link_libraries(endoscope /path/to/algo/lib/libalgo.so)
   # 运行时需要把 .so 部署到开发板的 /usr/lib/
   ```
4. 交叉编译注意事项：
   - 算法组提供的库必须是 ARM 架构编译的，不能用 x86 版本
   - 用 `file libalgo.a` 检查架构：应显示 `ELF 32-bit ARM`
   - glibc 版本必须兼容（和我们的 OpenCV/Qt 一样的问题）
5. 常见问题：
   - 头文件找不到 → 检查 `target_include_directories` 路径
   - 链接报错 undefined reference → 检查库文件路径和架构
   - 运行时找不到 .so → 检查 `LD_LIBRARY_PATH` 或复制到 `/usr/lib/`

**目标要求**：
- [ ] 能解释静态库和动态库的区别和优劣势
- [ ] 能在 CMakeLists.txt 中集成第三方 .a 或 .so 库
- [ ] 能排查库集成时的常见错误（架构不匹配、符号找不到、运行时加载失败）
- [ ] 理解交叉编译环境下集成第三方库的注意事项

**参考资源**：CMake 官方文档 https://cmake.org/cmake/help/latest/，项目文档 `docs/deploy/06_交叉编译项目.md`

### 2.4 V4L2

**学习方法**：逐行读 `v4l2capture.cpp`，对照 V4L2 API 文档理解每个 ioctl。

**学习路径**：
1. 设备打开：open("/dev/video0")
2. 能力查询：VIDIOC_QUERYCAP
3. 格式设置：VIDIOC_S_FMT（YUYV、分辨率）
4. Buffer 管理：VIDIOC_REQBUFS → VIDIOC_QUERYBUF → mmap
5. 采集循环：VIDIOC_QBUF → VIDIOC_DQBUF → 处理 → VIDIOC_QBUF
6. 流控制：VIDIOC_STREAMON / VIDIOC_STREAMOFF

**目标要求**：
- [ ] 能画出 V4L2 采集的完整流程图
- [ ] 能解释 mmap 的作用（用户空间直接访问内核 buffer，零拷贝）
- [ ] 能解释 YUYV 格式和 RGB 的区别
- [ ] 理解 buffer 队列机制（QBUF/DQBUF 轮转）

**参考资源**：V4L2 官方文档 https://www.kernel.org/doc/html/latest/userspace-api/media/v4l/v4l2.html
