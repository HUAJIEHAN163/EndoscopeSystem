# 04 — V4L2 摄像头采集

## 4.1 设计思路

将视频输入抽象为 `VideoSource` 接口，提供两个实现:

| 类 | 用途 | 运行环境 |
|----|------|---------|
| FileCapture | 读取视频文件，模拟摄像头 | 虚拟机开发 |
| V4l2Capture | V4L2 真实采集 | 开发板部署 |

程序启动时自动选择: 检测到 `/dev/video0` 用 V4L2，否则读测试视频。

---

## 4.2 VideoSource 抽象接口

```
文件: src/capture/videosource.h
```

核心设计:
- 继承 QThread，采集在独立线程运行
- `frameReady(QImage)` 信号传递帧数据
- `errorOccurred(QString)` 信号传递错误
- `std::atomic<bool> m_running` 线程安全的运行控制

所有使用视频源的代码只依赖 VideoSource 接口，不关心具体实现。

---

## 4.3 FileCapture — 视频文件输入

```
文件: src/capture/filecapture.h, filecapture.cpp
```

使用 OpenCV VideoCapture 读取视频文件:
- 支持 mp4/avi 等常见格式
- 视频结束自动循环播放
- fps 参数控制读取速度，模拟真实帧率
- Mat→QImage 转换后通过信号发出

虚拟机开发时，在 `config/endoscope.conf` 中配置测试视频路径:

```ini
[file]
path=test_data/test.mp4
```

## 4.4 V4l2Capture — V4L2 真实采集

```
文件: src/capture/v4l2capture.h, v4l2capture.cpp
```

### 借鉴来源

| 特性 | 来源 |
|------|------|
| QUERYCAP 能力查询 | cap-v4l2-master |
| ENUM_FMT 格式枚举 | cap-v4l2-master |
| TRY_FMT 格式试探 | cap-v4l2-master |
| select() 等待 | cap-v4l2-master |
| xioctl 重试封装 | cap-v4l2-master |
| QThread 封装 | VideoMonitor |
| 信号槽传递 QImage | VideoMonitor |
| V4L2 Controls | cap-v4l2-master (曝光/翻转) + 扩展 |

### V4L2 采集流程

```
open() 打开设备 (O_RDWR | O_NONBLOCK)
  ↓
queryCapability() — QUERYCAP 确认支持 Streaming
  ↓
enumFormats() — ENUM_FMT 打印支持的格式 (仅信息)
  ↓
tryAndSetFormat() — TRY_FMT + S_FMT 设置 YUYV 640x480
  ↓
setFrameRate() — S_PARM 设置 30fps
  ↓
requestBuffers() — REQBUFS + QUERYBUF + mmap + QBUF
  ↓
start() → run()
  ↓
startStreaming() — STREAMON
  ↓
循环: select() → DQBUF → cvtColor(YUYV→BGR→RGB) → QImage → emit → QBUF
  ↓
stop() → stopStreaming() — STREAMOFF
  ↓
close() → releaseBuffers() → munmap + close(fd)
```

### 与原 VideoMonitor V4L2 的区别

| 原 VideoMonitor | 本项目 |
|----------------|--------|
| 请求 V4L2_PIX_FMT_JPEG | 请求 V4L2_PIX_FMT_YUYV |
| 依赖 libjpeg 解码 | OpenCV cvtColor (有 NEON 优化) |
| 阻塞式 ioctl DQBUF | select() + DQBUF，带 2 秒超时 |
| 无能力查询 | QUERYCAP + ENUM_FMT + TRY_FMT |
| 无 V4L2 control | 12 个 control (曝光/增益/翻转/白平衡等) |
| 硬编码 640x480 | 配置文件驱动，TRY_FMT 自动调整 |
| volatile bool | std::atomic<bool> |
| 栈上分配 900KB buffer | OpenCV Mat 管理内存 |

### V4L2 Controls

代码中实现了 12 个硬件参数控制，对应 OV5640 实际支持的 controls:

```cpp
setExposure(int)          // 0-65535, 需先关闭自动曝光
setAutoExposure(bool)     // 开/关自动曝光
setGain(int)              // 0-1023, 需先关闭自动增益
setAutoGain(bool)         // 开/关自动增益
setContrast(int)          // 0-255
setSaturation(int)        // 0-255
setHue(int)               // 0-359
setHFlip(bool)            // 水平翻转
setVFlip(bool)            // 垂直翻转
setAutoWhiteBalance(bool) // 开/关自动白平衡
setRedBalance(int)        // 0-4095, 需先关闭自动白平衡
setBlueBalance(int)       // 0-4095, 需先关闭自动白平衡
```

## 4.5 条件编译

V4L2 代码仅在 Linux 上编译，通过 CMake 选项控制:

```cmake
option(ENABLE_V4L2 "Enable V4L2 camera capture (Linux only)" ON)
```

代码中用 `#ifdef ENABLE_V4L2` 保护，非 Linux 平台自动跳过。
