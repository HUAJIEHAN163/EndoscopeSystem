# 13_5 开发记录：三线程管线优化（P3）

## 一、背景

### 当前架构（两线程）

```
采集线程                              主线程
├── grabFrame                        ├── onFrameReady
├── cvtColor YUYV→BGR→RGB            │   ├── processFrame（算法）← 阻塞主线程
├── emit frameReady ──────────→      │   ├── update
└── 循环                             │   └── paintEvent（显示）
                                     ├── 按钮响应
                                     └── 其他 UI 事件
```

### 问题

1. 算法在主线程执行，阻塞 UI 刷新和按钮响应
2. 开 CLAHE（~28ms）+ 去雾（~87ms）时，主线程每帧耗时 ~130ms，FPS 降到 ~8
3. 算法设计被迫妥协性能（如锐化不能用 Lab 空间，因为多 20ms 不可接受）
4. m_pendingFrames 限制导致大量帧丢弃

```
*对第3点的详细说明
如果在 Lab 空间做锐化：
BGR → Lab          ~10ms
split              ~2ms
对 L 通道做 USM     ~7ms
merge              ~2ms
Lab → BGR          ~10ms
合计               ~31ms

当前在 BGR 空间直接做：
GaussianBlur       ~5ms
addWeighted        ~2ms
合计               ~7ms
多了 20ms 的颜色转换开销，在 Cortex-A7 上这个代价太大了。

效果区别：
BGR 空间锐化：对 B、G、R 三个通道同时锐化
  → 边缘处可能出现彩色伪影（颜色溢出）

Lab 空间锐化：只对 L（亮度）通道锐化，a、b 不动
  → 边缘更干净，不会出现彩色伪影

但在我们的场景下（320×240 降分辨率 + 内窥镜画面），BGR 锐化的彩色伪影不明显，7ms 和 31ms 的差距更重要。


CLAHE 为什么愿意付这个代价：
CLAHE 如果在 BGR 空间做，三个通道分别均衡化，颜色会严重失真——这是不可接受的。所以 CLAHE 必须在 Lab 空间做，颜色转换的开销是必须付的。
锐化在 BGR 空间做效果"还行"，所以选择省时间。这就是算法设计中的画质-性能权衡。
```

### 目标架构（三线程）

```
采集线程              处理线程（新增）          主线程
├── grabFrame         ├── processFrame        ├── paintEvent（只显示）
├── cvtColor          ├── CLAHE/去雾          ├── 按钮响应
├── push 到队列A ──→  ├── push 到队列B ──→    ├── 取队列B最新帧显示
└── 循环              └── 循环                └── 不再被算法阻塞
```

### 预期收益

| 指标 | 当前 | 优化后 |
|---|---|---|
| UI 响应 | 开算法时卡顿 | 始终流畅 |
| 显示 FPS | 受算法耗时限制（~8fps） | 接近采集帧率（~27fps） |
| 算法自由度 | 受性能妥协 | 可自由选择最优方案 |
| 延迟 | 算法耗时直接叠加 | 算法耗时变为延迟，不影响帧率 |

## 二、设计方案

### 2.1 数据流

```
采集线程：
  grabFrame → cvtColor(YUYV→BGR) → returnBuffer → cvtColor(BGR→RGB)
  → QImage copy → 写入 m_captureQueue（环形队列A）

处理线程（新增）：
  从 m_captureQueue 读取 → processFrame（CLAHE/去雾等）
  → 写入 m_displayQueue（环形队列B）

主线程：
  定时器/信号 → 从 m_displayQueue 读取最新帧 → paintEvent 显示
```

### 2.2 环形队列设计

单生产者单消费者，无锁设计：

```cpp
template<int N>
class FrameQueue {
    QImage frames[N];
    std::atomic<int> writeIdx{0};
    std::atomic<int> readIdx{0};
public:
    bool push(const QImage &frame);  // 生产者调用
    bool pop(QImage &frame);         // 消费者调用
```

### 5.1 Step 1：新增 FrameQueue 环形队列

新增文件：`src/utils/framequeue.h`

无锁环形队列，单生产者单消费者，用于线程间传递 QImage 帧数据。

设计要点：
- 固定大小数组（模板参数 N），不动态分配内存
- writeIdx 只有生产者写，readIdx 只有消费者写，用 atomic 保证可见性
- 队列满时覆盖最旧帧（移动 readIdx），保证生产者不阻塞
- 提供 latest() 方法，跳过中间帧只取最新帧（主线程显示用）

```cpp
#ifndef FRAMEQUEUE_H
#define FRAMEQUEUE_H

#include <QImage>
#include <atomic>

template<int N>
class FrameQueue {
    QImage m_frames[N];
    std::atomic<int> m_writeIdx{0};
    std::atomic<int> m_readIdx{0};

public:
    // 生产者调用：写入一帧
    // 队列满时覆盖最旧帧（移动 readIdx），保证生产者不阻塞
    void push(const QImage &frame) {
        int w = m_writeIdx.load(std::memory_order_relaxed);
        int next = (w + 1) % N;
        if (next == m_readIdx.load(std::memory_order_acquire)) {
            m_readIdx.store((m_readIdx.load(std::memory_order_relaxed) + 1) % N,
                            std::memory_order_release);
        }
        m_frames[w] = frame;
        m_writeIdx.store(next, std::memory_order_release);
    }

    // 消费者调用：读取一帧，返回 false 表示队列空
    bool pop(QImage &frame) {
        int r = m_readIdx.load(std::memory_order_relaxed);
        if (r == m_writeIdx.load(std::memory_order_acquire))
            return false;
        frame = m_frames[r];
        m_readIdx.store((r + 1) % N, std::memory_order_release);
        return true;
    }

    // 消费者调用：取最新帧，跳过中间所有帧
    // 用于主线程显示——只关心最新的处理结果
    bool latest(QImage &frame) {
        int r = m_readIdx.load(std::memory_order_relaxed);
        int w = m_writeIdx.load(std::memory_order_acquire);
        if (r == w) return false;
        int newest = (w - 1 + N) % N;
        frame = m_frames[newest];
        m_readIdx.store(w, std::memory_order_release);
        return true;
    }

    int size() const {
        int w = m_writeIdx.load(std::memory_order_relaxed);
        int r = m_readIdx.load(std::memory_order_relaxed);
        return (w - r + N) % N;
    }

    bool empty() const { return size() == 0; }
};

#endif // FRAMEQUEUE_H
```

### 5.2 Step 2：新增 ProcessThread 处理线程类

新增文件：`src/processing/processthread.h`

从 captureQueue 取帧 → 执行算法 → 写入 displayQueue。主线程从 displayQueue 取最新帧显示。

```cpp
#ifndef PROCESSTHREAD_H
#define PROCESSTHREAD_H

#include <QThread>
#include <QImage>
#include <atomic>
#include "utils/framequeue.h"
#include "utils/imageconvert.h"
#include "processing/imageprocessor.h"

class ProcessThread : public QThread {
    Q_OBJECT
public:
    explicit ProcessThread(FrameQueue<4> *captureQueue,
                           FrameQueue<4> *displayQueue,
                           QObject *parent = nullptr)
        : QThread(parent),
          m_captureQueue(captureQueue),
          m_displayQueue(displayQueue),
          m_running(false),
          m_quit(false) {}

    void pause() { m_running = false; }
    void resume() { m_running = true; }
    void stop() { m_quit = true; m_running = false; }

    // 算法配置（主线程通过 UI 控件更新，处理线程读取）
    ImageProcessor::Config config;

    // 畸变校正映射表（只读）
    cv::Mat undistortMap1, undistortMap2;

    // 显示区域尺寸
    int displayWidth = 640;
    int displayHeight = 480;

protected:
    void run() override {
        m_running = true;
        while (!m_quit) {
            if (!m_running) {
                QThread::msleep(10);
                continue;
            }

            QImage frame;
            if (!m_captureQueue->pop(frame)) {
                QThread::msleep(1);
                continue;
            }

            bool anyEnabled = config.clahe || config.undistort ||
                              config.dehaze || config.sharpen ||
                              config.denoise || config.edgeDetect ||
                              config.threshold;

            QImage result;
            if (!anyEnabled) {
                result = frame.scaled(displayWidth, displayHeight,
                                      Qt::KeepAspectRatio, Qt::FastTransformation);
            } else {
                cv::Mat src = ImageConvert::qimageToMat(frame);
                if (src.empty()) continue;
                cv::Mat dst;
                ImageProcessor::process(src, dst, config,
                                        undistortMap1, undistortMap2);
                cv::Mat display;
                cv::resize(dst, display, cv::Size(displayWidth, displayHeight),
                           0, 0, cv::INTER_LINEAR);
                result = ImageConvert::matToQImage(display);
            }

            m_displayQueue->push(result);
        }
    }

private:
    FrameQueue<4> *m_captureQueue;
    FrameQueue<4> *m_displayQueue;
    std::atomic<bool> m_running;
    std::atomic<bool> m_quit;
};

#endif // PROCESSTHREAD_H
```

```cpp
    m_source->m_captureQueue = &m_captureQueue;

    m_processThread->displayWidth = m_videoRect.width();
    m_processThread->displayHeight = m_videoRect.height();

    m_displayTimer = new QTimer(this);

            m_processThread->config.whiteBalance = false;
            m_processThread->config.clahe = m_chkClahe->isChecked();

            m_processThread->config.undistort = m_chkUndistort->isChecked() && m_undistortReady;

        } else if (m_processThread && m_clinicalMode) {
            m_processThread->config = m_procConfig;
        }

        QImage frame;
        if (m_displayQueue.latest(frame)) {
            m_latestRawFrame = frame;  // 保存原始帧用于拍照
            m_frameCount++;

            // 旋转
            if (m_rotateAngle != 0) {
                QTransform transform;
                transform.rotate(m_rotateAngle);
                frame = frame.transformed(transform, Qt::FastTransformation);
                frame = frame.scaled(m_videoRect.size(), Qt::KeepAspectRatio,
                                     Qt::FastTransformation);
            }

            // 录像
            if (m_recording && m_videoWriter.isOpened()) {
                cv::Mat mat = ImageConvert::qimageToMat(frame);
                if (!mat.empty()) {
                    cv::Mat resized;
                    cv::resize(mat, resized,
                               cv::Size(m_videoRect.width(), m_videoRect.height()));
                    m_videoWriter.write(resized);
                }
            }

            m_displayImage = frame;
            update();
        }
    });
    m_displayTimer->start(16);

    // 启动采集线程和处理线程
    m_source->start();
    m_processThread->start();

    // 错误信号
```

### 关键变化

| 项目 | 改动前 | 改动后 |
|---|---|---|
| 线程间通信 | Qt 信号槽（QueuedConnection） | 无锁环形队列 |
| 算法执行位置 | 主线程 | 独立处理线程 |
| 主线程职责 | 算法 + 显示 + UI | 只显示 + UI |
