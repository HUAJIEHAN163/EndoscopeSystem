# 13_4 开发记录：实时处理与离线处理分离

## 一、背景

### 当前问题

所有图像处理算法（白平衡、CLAHE、去雾、锐化、降噪、边缘检测、阈值分割）都在实时视频流中执行，导致：

1. 开多个算法后 FPS 严重下降（CLAHE + 锐化 + 降噪 → 个位数 FPS）
2. 实时流中跑重算法没有必要——医生实时观察只需要基础增强
3. 锐化、降噪等精细处理应该在拍照后对静态图做，效果更好、不受帧率限制

### 实际内窥镜的工作流

```
实时观察（边看边做）          拍照后处理（存下来再做）
├── 硬件白平衡/曝光（OV5640） ├── 锐化（精细，不限时间）
├── CLAHE 增强（看清暗区）    ├── 降噪（高质量双边滤波）
├── 去雾（水雾环境）          ├── 边缘检测（辅助诊断）
└── 显示 + 拍照键             ├── 阈值分割（辅助分析）
         ↓                    ├── 测量、标注（后续规划）
    保存原图（全分辨率）       └── 导出报告（后续规划）
```

## 二、目标架构

### 2.1 三个主要界面

```
┌─────────────────────────────────────────────────────┐
│  [实时采集]    [图库浏览]    [图像编辑]               │
├─────────────────────────────────────────────────────┤
│                                                     │
│              当前选中的界面内容                       │
│                                                     │
├─────────────────────────────────────────────────────┤
│  [拍照]  [录像]  [冻结]  [旋转]     FPS: 20.0      │
└─────────────────────────────────────────────────────┘
```

**实时采集页**（当前主界面，精简后）：
- 视频画面
- 硬件参数：自动白平衡、自动曝光、翻转
- 实时算法：CLAHE、去雾（只保留轻量算法）
- 操作：拍照、录像、冻结、旋转

**图库浏览页**（新增）：
- 缩略图列表，显示 captures/ 下的所有图片
- 点击缩略图 → 全屏查看
- 长按或按钮 → 进入编辑模式
- 删除功能

**图像编辑页**（新增）：
- 显示选中的静态图片（全分辨率）
- 离线算法：锐化、降噪、边缘检测、阈值分割
- 参数滑块实时预览（对静态图处理，不受帧率限制）
- 保存/另存为

### 2.2 数据流

```
实时采集页：
  OV5640 → 硬件白平衡/曝光 → CLAHE/去雾 → 显示
                                          ↓ 拍照
                                    保存原图到 captures/
                                          ↓
图库浏览页：
  读取 captures/ 目录 → 缩略图列表 → 点击查看
                                          ↓ 编辑
图像编辑页：
  加载原图 → 锐化/降噪/边缘检测/阈值分割 → 预览 → 保存
```

### 2.3 拍照保存策略

拍照时保存**原始全分辨率图像**（640x480，未经软件处理），而不是显示尺寸的处理后图像：

```cpp
// 当前（保存显示图像，分辨率低，已处理）
m_displayImage.save(filename, "JPEG", 95);

// 改为（保存原始采集帧，全分辨率，未经软件处理）
m_latestRawFrame.save(filename, "JPEG", 95);
```

这样离线编辑时有最大的处理空间。
