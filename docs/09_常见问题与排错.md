# 09 — 常见问题与排错

## 9.1 构建相关

### Q: cmake 找不到 Qt5

```
Could not find a package configuration file provided by "Qt5"
```

**解决**: 安装 Qt5 开发包:
```bash
sudo apt-get install qtbase5-dev
```

交叉编译时需要指定路径:
```bash
cmake .. -DQt5_DIR=/path/to/qt5_arm/lib/cmake/Qt5
```

### Q: cmake 找不到 OpenCV

```
Could not find a package configuration file provided by "OpenCV"
```

**解决**: 安装 OpenCV 开发包:
```bash
sudo apt-get install libopencv-dev
```

交叉编译时:
```bash
cmake .. -DOpenCV_DIR=/path/to/opencv_arm/lib/cmake/opencv4
```

### Q: Git 报 "dubious ownership" 错误

```
fatal: detected dubious ownership in repository
```

**原因**: 项目在 VMware 共享文件夹中，文件所有者与当前用户不一致。

**解决**:
```bash
git config --global --add safe.directory /mnt/hgfs/VM_Share/project/EndoscopeSystem
```

## 9.2 摄像头相关

### Q: /dev/video0 不存在

1. 检查设备树 overlay 是否启用:
```bash
cat /boot/uEnv.txt | grep cam
# 确认 stm-fire-cam.dtbo 未被注释
```

2. 检查内核日志:
```bash
dmesg | grep -i "ov5640\|dcmi"
```

3. 检查硬件连接 (排线方向、是否插紧)

### Q: V4L2 DQBUF 超时

**原因**: select() 2 秒内没有帧就绪。

**排查**:
```bash
# 确认摄像头能正常抓帧
v4l2-ctl --device=/dev/video0 \
    --set-fmt-video=width=640,height=480,pixelformat=YUYV \
    --stream-mmap=4 --stream-count=1 --stream-to=/tmp/test.raw
```

### Q: 画面绿屏/花屏

**原因**: 像素格式不匹配。

**排查**:
```bash
v4l2-ctl --device=/dev/video0 --get-fmt-video
# 确认实际格式与代码中的 cvtColor 参数一致
```

代码中使用 `cv::COLOR_YUV2BGR_YUYV`，如果实际格式是 UYVY，
需要改为 `cv::COLOR_YUV2BGR_UYVY`。

## 9.3 Qt 显示相关

### Q: 开发板上 "Could not find the Qt platform plugin linuxfb"

```bash
# 设置插件路径
export QT_QPA_PLATFORM_PLUGIN_PATH=/usr/lib/qt5/plugins
export QT_QPA_PLATFORM=linuxfb:fb=/dev/fb0
```

### Q: 虚拟机上运行时窗口不显示

确认虚拟机有图形桌面环境。如果是纯命令行系统，需要安装桌面:
```bash
sudo apt-get install ubuntu-desktop
```

## 9.4 图像处理相关

### Q: CLAHE 处理后颜色异常

**原因**: QImage→Mat 转换时颜色空间不对。

**排查**: ImageConvert::qimageToMat 中 RGB888 格式会做 `COLOR_RGB2BGR` 转换，
确认输入 QImage 的 format 是否正确。

### Q: 去雾算法太慢

**优化方案** (按优先级):
1. 减小 radius: 7 → 5
2. 降采样处理: 缩小到一半处理再放大
3. 隔帧处理: 每 2 帧处理一次
4. 不与降噪同时开启

### Q: 畸变校正 checkbox 灰色不可用

**原因**: 程序启动时未找到 `camera_calib.yml` 标定文件。

**解决**: 需要先完成镜头标定 (Phase 3)。

## 9.5 运行时相关

### Q: 开发板上 "error while loading shared libraries"

```bash
# 查看缺少哪个库
ldd /opt/endoscope/endoscope

# 拷贝缺少的库
scp /opt/opencv_arm/lib/libopencv_*.so* root@192.168.0.43:/usr/lib/
ssh root@192.168.0.43 "ldconfig"
```

### Q: 帧率很低

**排查**:
1. 检查同时开启了哪些算法 (去雾+降噪 很慢)
2. 检查分辨率 (降到 320x240 测试)
3. `top` 查看 CPU 占用
4. 确认 OpenCV 编译时开启了 NEON
