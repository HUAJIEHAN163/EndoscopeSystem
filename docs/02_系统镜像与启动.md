## 2.2 网络配置

开发板通过 WiFi 连接路由器：

| 设备 | IP | 连接方式 |
|------|-----|---------|
| 开发板 | 192.168.0.43 | WiFi (wlan0, TP-LINK_CAAD) |
| Windows 主机 | 192.168.0.10 | 有线 |
| VMware 虚拟机 | 192.168.65.x 或 192.168.40.x | VMnet1/VMnet8 |

> **注意**: 虚拟机如果使用 NAT 模式 (VMnet8)，与开发板不在同一网段，
> 需要改为桥接模式才能直接通信。后续部署时需要确认网络互通。

## 2.3 摄像头状态

### 驱动信息

```
Driver:    stm32-dcmi
Card:      STM32 Camera Memory Interface
Bus:       platform:dcmi
能力:      Video Capture, Read/Write, Streaming
```

摄像头通过设备树 overlay 启用 (`stm-fire-cam.dtbo` 在 `/boot/uEnv.txt` 中取消注释)。

内核已编译 DCMI 和 OV5640 驱动 (`CONFIG_VIDEO_STM32_DCMI=y`, `CONFIG_VIDEO_OV5640=y`)。

### 支持的像素格式

| 格式 | V4L2 宏 | 说明 | 本项目使用 |
|------|---------|------|-----------|
| JPEG | V4L2_PIX_FMT_JPEG | 硬件压缩 | 否 |
| UYVY | V4L2_PIX_FMT_UYVY | YUV422 | 否 |
| **YUYV** | **V4L2_PIX_FMT_YUYV** | **YUV422** | **✅ 首选** |
| RGBP | V4L2_PIX_FMT_RGB565 | 16-bit RGB | 否 |

选择 YUYV 的原因:
- 无需 libjpeg 解码，减少依赖和 CPU 开销
- OpenCV cvtColor(COLOR_YUV2BGR_YUYV) 直接转换，有 NEON 优化
- 无损数据，适合后续图像处理

### 支持的分辨率与帧率

| 分辨率 | 帧率 | 说明 |
|--------|------|------|
| 176x144 | 15/30 fps | QCIF |
| 320x240 | 15/30 fps | QVGA |
| **640x480** | **15/30/60 fps** | **✅ 推荐** |
| 720x480 | 15/30 fps | NTSC |
| 1024x768 | 15/30 fps | XGA |
| 1280x720 | 15/30 fps | 720p |
| 1920x1080 | 15/30 fps | 1080p |
| 2592x1944 | 15/30 fps | 5MP |

推荐: **640x480 @30fps** — 匹配 800x480 屏幕，留出 CPU 余量给图像处理。

### V4L2 Controls

| Control | 范围 | 默认 | flags | 内窥镜用途 |
|---------|------|------|-------|-----------|
| contrast | 0-255 | 0 | slider | 对比度调节 |
| saturation | 0-255 | 64 | slider | 饱和度调节 |
| hue | 0-359 | 0 | slider | 色调调节 |
| exposure | 0-65535 | auto | inactive, volatile | 曝光控制 |
| gain | 0-1023 | auto | inactive, volatile | 增益控制 |
| horizontal_flip | 0/1 | 0 | | 水平翻转 |
| vertical_flip | 0/1 | 0 | | 垂直翻转 |
| white_balance_automatic | 0/1 | 1 | update | 自动白平衡 |
| red_balance | 0-4095 | 0 | inactive, slider | 手动红色平衡 |
| blue_balance | 0-4095 | 0 | inactive, slider | 手动蓝色平衡 |

> **重要**: exposure 和 gain 的 flags 为 `inactive`，需要先关闭自动模式才能手动设置。
