## 一、NEON 指令集加速

ARM Cortex-A7 支持 NEON SIMD 指令，可对图像处理关键路径进行向量化加速。

### 1.1 优化目标

| 模块 | 当前实现 | 优化方案 | 预期提升 |
|------|---------|---------|---------|
| YUYV → RGB 转换 | OpenCV cvtColor | NEON intrinsic 手写 | 3-5x |
| CLAHE 增强 | OpenCV CLAHE | 直方图统计 NEON 化 | 2-3x |
| 高斯滤波 | OpenCV GaussianBlur | NEON 定点化卷积 | 2-4x |

### 1.2 实施步骤

1. 在 `src/processing/` 下新建 `neon_accel.h/cpp`
2. 先实现 YUYV → RGB 的 NEON 版本，作为入门练习
3. 用 `arm_neon.h` 提供的 intrinsic 函数（vld1q_u8、vmull_u8 等）
4. 编写性能对比测试，验证加速效果
5. 逐步替换其他热点函数

### 1.3 示例：YUYV → RGB NEON 转换

```cpp
#include <arm_neon.h>

// 每次处理 16 个像素（32 字节 YUYV → 48 字节 RGB）
void yuyv_to_rgb_neon(const uint8_t* src, uint8_t* dst, int pixel_count);
```

### 1.4 编译配置

```cmake
# CMakeLists.txt 中添加 NEON 编译选项
target_compile_options(endoscope PRIVATE -mfpu=neon -mfloat-abi=hard)
```

## 二、实时性保障

确保 640x480@30fps 稳定运行，不丢帧、不卡顿。

### 2.1 线程架构优化

当前：采集和处理在同一个 QThread 中串行执行。

优化为三级流水线：

```
采集线程 → [环形队列] → 处理线程 → [环形队列] → 渲染线程(主线程)
```

#### 环形 Buffer 队列

```cpp
// src/utils/framebuffer.h
template<int N>
class FrameRingBuffer {
    cv::Mat frames[N];
    std::atomic<int> writeIdx{0};
    std::atomic<int> readIdx{0};
public:
    bool push(const cv::Mat& frame);  // 采集线程写入
    bool pop(cv::Mat& frame);         // 处理线程读取
};
```

- 固定 3-4 个 buffer 槽位
- 无锁设计，采集线程和处理线程各操作一端
- 队列满时丢弃最旧帧，保证实时性

#### 线程优先级设置

```cpp
// 采集线程设置较高优先级，确保不丢帧
#include <pthread.h>

struct sched_param param;
param.sched_priority = 80;
pthread_setschedparam(pthread_self(), SCHED_FIFO, &param);
```

### 2.2 内存池

避免每帧动态分配内存，预分配固定数量的 cv::Mat 循环复用。

```cpp
// src/utils/mempool.h
class FrameMemPool {
    std::vector<cv::Mat> pool;
    std::queue<int> freeList;
public:
    void init(int count, int rows, int cols, int type);
    cv::Mat& acquire();   // 获取空闲帧
    void release(int id); // 归还帧
};
```

### 2.3 性能监控

添加帧率统计和延迟监控，便于定位瓶颈：

```cpp
// 每秒统计一次实际帧率和单帧处理耗时
// 采集延迟 = 帧到达时间 - 上一帧到达时间
// 处理延迟 = 处理完成时间 - 帧到达时间
// 端到端延迟 = 显示时间 - 采集时间
```

## 四、A7 Linux 用户空间外设通信

在 Linux 应用层通过设备节点操作 UART/SPI/I2C，不涉及内核驱动开发。

### 4.1 I2C（读写传感器寄存器）

```cpp
// 通过 /dev/i2c-x 读写外设
#include <linux/i2c-dev.h>
#include <sys/ioctl.h>

int fd = open("/dev/i2c-1", O_RDWR);
ioctl(fd, I2C_SLAVE, 0x3C);  // 设置从机地址
write(fd, data, len);         // 写寄存器
read(fd, buf, len);           // 读寄存器
```

练习方向：挂一个 OLED (SSD1306) 或温湿度传感器 (SHT30)，用 I2C 驱动显示数据。

### 4.2 SPI（高速外设通信）

```cpp
// 通过 /dev/spidev0.0 操作
#include <linux/spi/spidev.h>

int fd = open("/dev/spidev0.0", O_RDWR);
struct spi_ioc_transfer tr = { .tx_buf = tx, .rx_buf = rx, .len = len, .speed_hz = 1000000 };
ioctl(fd, SPI_IOC_MESSAGE(1), &tr);
```

练习方向：驱动一个 SPI 接口的 TFT 小屏或 Flash 芯片。

### 4.3 UART（串口通信）

```cpp
// 通过 /dev/ttySTM0 等串口设备
#include <termios.h>

int fd = open("/dev/ttySTM0", O_RDWR | O_NOCTTY);
struct termios tty;
tcgetattr(fd, &tty);
cfsetospeed(&tty, B115200);
tcsetattr(fd, TCSANOW, &tty);
write(fd, "hello", 5);
```

练习方向：A7 与 M4 之间通过 UART 通信，或连接外部 MCU/模块。

## 五、M4 裸机开发（CubeIDE + HAL 库）

STM32MP157 的 Cortex-M4 侧，使用 CubeIDE 进行裸机/RTOS 开发，这是嵌入式开发的基本功。

### 5.1 开发环境

- IDE：STM32CubeIDE
- 配置工具：CubeMX（引脚配置、时钟树、外设初始化代码生成）
- 调试：ST-Link + SWD
- 语言：C

### 5.2 核心学习内容

| 模块 | 内容 | 说明 |
|------|------|------|
| GPIO | LED 控制、按键输入 | HAL 入门第一步 |
| UART | 串口收发、printf 重定向 | 调试必备 |
| I2C | 传感器读写（温湿度、IMU） | 掌握主从通信时序 |
| SPI | Flash 读写、显示屏驱动 | 理解全双工高速通信 |
| TIM | 定时器中断、PWM 输出 | 电机控制、LED 调光 |
| DMA | 数据搬运不占 CPU | 配合 UART/SPI/ADC 使用 |
| ADC | 模拟信号采集 | 电压、温度采集 |
| 中断 | NVIC 优先级管理 | 实时响应的基础 |

### 5.3 进阶方向

- FreeRTOS：多任务调度、信号量、消息队列
- A7-M4 核间通信：OpenAMP / RPMsg，A7 下发指令，M4 实时执行
- 低功耗管理：Stop/Standby 模式

### 5.4 与本项目的结合

```
A7 (Linux + Qt)                    M4 (裸机/FreeRTOS)
┌──────────────┐    RPMsg/UART    ┌──────────────────┐
│ 图像采集显示  │ ◄────────────► │ LED 补光灯 PWM    │
│ 算法处理      │                │ 电机控制(调焦)    │
│ 用户界面      │                │ 温度传感器监控    │
└──────────────┘                └──────────────────┘
```

内窥镜产品中，M4 常负责：
- LED 冷光源亮度调节（PWM）
- 步进电机调焦控制
- 温度监控（防止探头过热）
- 按键/脚踏开关的实时响应
