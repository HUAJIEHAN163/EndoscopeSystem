# 07 — 功能集成与测试

## 7.1 数据流总览

```
采集线程                          GUI 线程
┌──────────┐  frameReady(QImage)  ┌──────────────────┐
│VideoSource│ ──────────────────→ │ onFrameReady()   │
│ V4L2/File │                     │   ↓              │
└──────────┘                     │ processFrame()   │
                                  │   ↓ (如果有算法)  │
                                  │ ImageProcessor   │
                                  │   ↓              │
                                  │ 旋转 (如果需要)   │
                                  │   ↓              │
                                  │ VideoWriter (录像)│
                                  │   ↓              │
                                  │ scaled → update()│
                                  │   ↓              │
                                  │ paintEvent()     │
                                  │   绘制画面 + OSD  │
                                  └──────────────────┘
```

## 7.2 集成步骤

### 第一步: 跑通采集 + 显示 (不加图像处理)

在虚拟机上用测试视频验证:
- FileCapture 能读取视频
- 画面在 Qt 窗口中实时显示
- 颜色正确，帧率正常

### 第二步: 逐个加入算法

每加一个算法测试一次，确认效果和帧率:

```
1. 白平衡 → 测试
2. CLAHE → 测试
3. 锐化 → 测试
4. 降噪 → 测试
5. 边缘检测 → 测试
6. 阈值分割 → 测试
7. 去雾 → 测试
8. 畸变校正 → 测试 (需标定文件)
```

### 第三步: 加入拍照/录像/冻结/旋转

## 7.3 功能测试用例

### 基础功能

| 测试项 | 操作 | 预期结果 |
|--------|------|---------|
| 启动 (有视频) | 运行程序 | 显示测试视频画面 |
| 启动 (无视频无摄像头) | 运行程序 | 状态栏显示 "无视频源"，不崩溃 |
| 拍照 | 按 C 键 | ./captures/ 下生成 JPG |
| 录像开始 | 按 R 键 | 按钮变 "停止录像"，OSD 显示红点 |
| 录像停止 | 再按 R | ./videos/ 下生成 AVI |
| 冻结 | 按 Space | 画面停止更新 |
| 解冻 | 再按 Space | 画面恢复 |
| 旋转 | 按 T | 画面旋转 90° |
| 退出 | 按 ESC | 正常退出 |

### 图像处理

| 测试项 | 操作 | 预期结果 |
|--------|------|---------|
| 单算法 | 勾选 CLAHE | 画面对比度增强 |
| 参数调节 | 拖动 CLAHE 滑块 | 效果实时变化 |
| 多算法组合 | 勾选 CLAHE + 锐化 | 两种效果叠加 |
| 全部关闭 | 取消所有勾选 | 画面恢复原始 |
| 畸变校正 (无标定) | 勾选畸变校正 | checkbox 不可用 (灰色) |

### 异常测试

| 测试项 | 预期结果 |
|--------|---------|
| 无摄像头启动 | 降级为文件模式或显示错误，不崩溃 |
| 长时间运行 (30分钟) | 内存不持续增长 |
| 快速切换算法 | 不崩溃，不花屏 |
| 录像中拍照 | 两者互不影响 |
| 冻结中拍照 | 保存冻结的画面 |
